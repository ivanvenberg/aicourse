**Кто делает:** пара вместе
**Дедлайн:** до следующей субботы
**Результат:** бот с личными кабинетами задеплоен по ссылке

---

## Что строим

Приложение с личным кабинетом и ботом которого вы сами придумали.

**Как работает:**
1. Пользователь вводит имя → попадает в свой кабинет
2. В кабинете — диалог с ботом
3. История сохраняется в Supabase
4. Вернулся через неделю — история на месте

---

## Шаг 1. Выберите бота

Договоритесь в паре — кого хотите построить:

| Идея | Описание |
|------|---------|
| 🎭 Персонаж | Шерлок Холмс, Стив Джобс, Йода — любой |

| 🎮 Игра | Детектив, текстовое приключение, угадайка |
| 💼 Помощник | По вашей профессии — бот-редактор, бот-бриф |
| 🤪 Что угодно | Ваша идея — главное чтобы интересно |

Запишите: «Наш бот — это [кто], он [что делает]»

---

## Шаг 2. Напишите системный промпт

Системный промпт — инструкция для бота. Кто он, как говорит.

Пример для бота-Шерлока:
```
Ты Шерлок Холмс. Говоришь коротко и немного надменно.
Задаёшь уточняющие вопросы. Делаешь неожиданные выводы.
Иногда упоминаешь Ватсона. Никогда не выходишь из роли.
```

Пример для коуча:
```
Ты коуч который помогает разобраться в рабочих проблемах.
Не даёшь советов — только задаёшь вопросы.
Один вопрос за раз. После 5-6 вопросов помогаешь сформулировать вывод.
```

## Если что-то не работает

**Вход не работает:**
```
После ввода имени и нажатия кнопки ничего не происходит.
Вот ошибка из консоли (F12): [вставь].
Почини.
```

**Бот не держит роль:**
```
Бот отвечает как обычный ассистент, не в роли.
Вот системный промпт: [вставь].
Убедись что промпт передаётся в каждый запрос к API.
```

**История пропадает:**
```
После перезагрузки история исчезает.
Должна сохраняться в Supabase таблице messages.
Посмотри почему не сохраняется и почини.
```

**Чужие сообщения видны:**
```
Пользователь видит сообщения других.
Нужно фильтровать по username в запросе к Supabase.
Почини.
```

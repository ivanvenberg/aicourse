## Шаг 1. Сначала сделай рабочий чат (Сценарий Б)

Если ничего нет или не работает — не пытайся чинить по кусочкам. Начни заново с чистого промпта.

Открой Claude Code в папке проекта. Вставь:

```
Создай простое веб-приложение — чат между двумя людьми.

Supabase URL: [вставь свой URL из supabase.com → Settings → API]
Supabase anon key: [вставь свой ключ]

Таблица называется messages, в ней поля:
- id (генерируется автоматически)
- username (text) — имя отправителя
- text (text) — текст сообщения
- created_at (timestamp) — время

Что должно работать:
1. При открытии спрашивает имя пользователя (сохраняет в localStorage)
2. Показывает все сообщения из базы
3. Поле ввода + кнопка отправить
4. Свои сообщения — справа, чужие — слева
5. Обновляет сообщения каждые 2 секунды автоматически
6. Тёмный дизайн, похоже на Telegram

Сохрани в папку product/index.html
```

После — открой в браузере, проверь что работает. Напиши что-нибудь, обнови страницу — сообщение должно остаться.

## Шаг 2. Почини логин (Сценарий В)

Если после ввода имени ничего не происходит — открой консоль браузера:

1. Нажми **F12** (или правая кнопка → Просмотр кода)
2. Перейди на вкладку **Console**
3. Скопируй весь красный текст

Вставь в Claude Code:
```
После ввода имени и нажатия кнопки — ничего не происходит.
Вот ошибки из консоли браузера:
[вставь скопированный текст]

Вот что должно происходить: после ввода имени — 
появляется чат где можно писать сообщения.

Почини.
```

Если ошибок нет в консоли — вставь другой промпт:
```
После ввода имени и нажатия кнопки ничего не происходит,
ошибок в консоли нет.
Посмотри код и найди почему кнопка не реагирует.
Скорее всего проблема в обработчике события или в роутинге между экранами.
Почини.
```

## Шаг 3. Добавь @ai в чат

Когда чат работает — добавляем AI.

Тебе нужен API ключ. Варианты:
- Зарегистрируйся на [openrouter.ai](https://openrouter.ai) → Keys → Create (бесплатная регистрация, платишь за использование)
- Или возьми общий ключ у организатора

Вставь в Claude Code:
```
Добавь в чат AI-ассистента.

API ключ OpenRouter: [вставь ключ]
Модель: google/gemini-flash-1.5

Логика:
- Если сообщение начинается с @ai — отправь его в AI
- Убери @ai из начала перед отправкой
- Ответ AI сохрани в базу как сообщение от username «🤖 AI»
- Остальные сообщения работают как раньше

Важно: API ключ должен быть в коде на клиенте
(это демо, для продакшна нужен бэкенд — об этом позже).
```

**Проверка:** напиши в чате `@ai привет, кто ты?` — должен ответить.

## Если что-то не получается

**Ошибка «Failed to fetch» или «CORS»:**
```
При отправке сообщения в консоли ошибка [вставь ошибку].
Почини.
```

**Supabase не подключается:**
```
Supabase возвращает ошибку [вставь ошибку].
Проверь что URL и ключ правильные.
Вот мои данные: URL = [твой URL], ключ = [первые 10 символов ключа]...
```

**AI не отвечает:**
```
@ai не работает, ошибка в консоли: [вставь].
OpenRouter ключ точно правильный.
Почини запрос к API.
```

**Совсем ничего не понимаю:**
```
Посмотри весь мой код. Объясни что делает каждая часть простыми словами.
Потом скажи что сломано и как починить.
```

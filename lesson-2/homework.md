**Кто делает:** пара вместе
**Дедлайн:** до следующей субботы
**Результат:** Telegram-подобный групповой чат с AI-ботом

---

## Что строим

Групповой чат где видны сообщения всех участников — как Telegram. Только вместо случайных людей — вы двое и бот которого вы придумали.

**Как работает:**
1. Вводишь имя → входишь в общий чат
2. Видишь сообщения всех кто когда-либо писал
3. Пишешь сообщение → все видят
4. Бот участвует в разговоре

---

## Шаг 3. Попросите Claude Code построить это

```
Создай Telegram-подобный групповой чат с AI-ботом.

Supabase URL: [вставь]
Supabase anon key: [вставь]
OpenRouter API key: [вставь — или попроси у организатора]
Модель: google/gemini-flash-1.5

Как работает:
1. Экран входа: поле «Введи имя» + кнопка войти
2. Имя сохраняется в localStorage
3. После входа — общий чат
4. Таблица messages: id, username, role (user/bot), text, created_at
5. Видны ВСЕ сообщения от ВСЕХ пользователей — как в групповом чате
6. Своё имя слева вверху, кнопка «Выйти»
7. Бот отвечает на каждое сообщение пользователя
8. Сообщения пользователей — справа, бота — слева
9. Обновление каждые 2 секунды

Системный промпт для бота:
[вставь ваш промпт]

Дизайн: тёмный, как Telegram.
```

## Если что-то не работает

**Вход не работает:**
```
После ввода имени ничего не происходит.
Ошибка в консоли (F12): [вставь].
Почини.
```

**Не видны чужие сообщения:**
```
Пользователь видит только свои сообщения.
Должны быть видны сообщения всех пользователей.
Убери фильтр по username из запроса к Supabase.
```

**Бот не держит роль:**
```
Бот отвечает не в роли.
Системный промпт: [вставь].
Убедись что промпт передаётся в каждый запрос.
```

**История пропадает:**
```
После перезагрузки история исчезает.
Должна загружаться из Supabase при открытии чата.
Почини.
```

## Ещё одно. Выберите проект

Параллельно с чатом — договоритесь в паре что будете строить на протяжении всего курса.

Это продукт который вы доведёте до конца. Не учебный пример — то чем реально хочется пользоваться.

**Вопрос:**
> Что мы делаем руками каждую неделю — и что нас в этом бесит?

Найдите пересечение между вашими профессиями. Лучшие идеи — на стыке.

**Примеры:**
- Дизайнер + маркетолог → инструмент для быстрого брифа с автогенерацией идей
- Редактор + исследователь → база знаний которая сама находит нужное
- Урбанист + дизайнер → инструмент для визуализации концепции объекта

Запишите одним предложением и скиньте в чат вместе со ссылкой на бота.

На следующей встрече начнём строить именно это.

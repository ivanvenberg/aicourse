# Урок 2. Первый живой продукт

**Сегодня каждая пара деплоит живой чат с базой данных и AI внутри.**

> Пары объявляются в начале. Роли: Строитель и Публикатор. Меняетесь в домашке.

---

## Роли

**🔨 Строитель** — пишет PRD, строит продукт, добавляет фичи, пушит в GitHub.
**🚀 Публикатор** — создаёт GitHub репо, подключает Vercel, настраивает Supabase, деплоит.

---

## До встречи (попросить сделать заранее)

- Создать аккаунт на github.com
- Создать аккаунт на supabase.com
- Создать аккаунт на openrouter.ai → получить API ключ
- Создать GitHub Personal Access Token: Settings → Developer Settings → Tokens (classic) → галочка `repo`

✅ Оба установили → написали + в чат

## Задание 3. GitHub + Vercel (15 мин)

**Публикатор:**
1. Создаёт репозиторий на github.com
2. Даёт ссылку Строителю

**Строитель** в Claude Code:
```
Инициализируй git репозиторий.
Сделай первый коммит.
Запушь на GitHub: [ссылка на репо]
Токен: [GitHub токен]
```

**Публикатор:**
3. Заходит на vercel.com → New Project → импортирует репозиторий с GitHub
4. Deплоит → копирует ссылку

Теперь каждый `git push` = автодеплой. Больше не надо деплоить руками.

✅ Ссылка на живой сайт есть → написали в чат

## 🔥 Задание 5: Псевдотелеграм (30 мин)

Строим настоящий живой чат между двумя людьми.

**Публикатор — настраивает Supabase:**

1. supabase.com → New Project
2. Table editor → New table → называем `messages`
3. Колонки: `id` (auto), `username` (text), `text` (text), `created_at` (timestamp)
4. Settings → API → копирует URL и anon key → передаёт Строителю

**Строитель — строит чат:**
```
Создай чат-приложение.
Supabase URL: [вставь]
Supabase anon key: [вставь]
Таблица: messages (id, username, text, created_at)

Придумайте название чата и два никнейма.
Один пишет → второй видит → отвечает.
Обновление каждые 2 секунды.
Дизайн: тёмный, как Telegram.
```

`git push` → Публикатор видит деплой → оба открывают ссылку на разных устройствах.

**Проверка:** один пишет — другой видит без перезагрузки.

✅ Чат работает между двумя устройствами → написали + в чат

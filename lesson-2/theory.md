## 0. Сначала — как думает человек

Прежде чем говорить о нейросетях, важно понять от чего они отличаются.

**Гештальт-принцип:** целое — не сумма частей.

Едешь в поезде ночью — видишь не отдельные фонари, а одну светящуюся линию. Мозг сам собирает разрозненное в единый образ. Так же работает слово «кошка» — ты не перебираешь лапы, уши, хвост. Ты сразу видишь кошку целиком.

**Человек думает от общего к частному.** Сначала образ — потом детали. Скажи «кошка» — она уже есть в голове. Полная. Не идеальная, но узнаваемая.

**AI думает от частного к общему.** Сначала пиксели — потом линии — потом формы — потом части — потом «кошка». Группировать ему сложно. Разбивать на части — просто.

Именно поэтому человеку сложно объяснить задачу от мелочей к целому — мы так не думаем. AI именно так и работает. Учти это когда объясняешь ему что строить: давай образ целиком, а не перечень пикселей.

## 1.1 Что такое нейросеть

Нейросеть — машина для последовательного преобразования данных через слои.

Берём фото кошки. На входе — пиксели. Просто числа, ничего больше.

Дальше слои:

- **Слой 1** → простые признаки (края, контрасты)
- **Слой 2** → формы (кружки, силуэты)
- **Слой 3** → части (ухо, хвост, мордочка)
- **Выходной** → ответ (кошка 97%, собака 2%, грейпфрут 1%)

Никто не говорит слоям что искать — они находят это сами.

**Ключевая штука: веса.** Не все пиксели одинаково важны. В процессе обучения каждому признаку присваивается вес — множитель от 0 до ~2. Высокий вес у глаз кошки. Низкий у фона. Глаза важны — без них как будто и не кошка. Фон — неважен.

Обучение = долгий перебор весов пока ошибка не станет минимальной. Нет никакой магии — просто очень много итераций.

## 1.1.1 История: откуда взялся ChatGPT

Нейросети — не вчера придумали. Первые появились в 90-х. Поиск по картинкам в Яндексе — та же технология, только старая и ограниченная.

Долго не взлетало по одной причине: не хватало вычислительной мощности.

Потом пришла NVIDIA. Видеокарты — изначально для игр. Там нужно быстро перемножать огромные матрицы чисел. Оказалось, нейросетям нужно то же самое. GPU подошёл идеально.

Это не стратегия — случайность. NVIDIA просто оказалась в нужном месте в нужное время. Теперь Anthropic, OpenAI и все остальные платят им огромные деньги за железо.

## 1.2 Трансформер и языковая модель

**GPT** = Generative Pre-trained Transformer.

Самое важное слово — **трансформер**.

Языковая модель — это **функция предсказания следующего токена**.

Не база знаний. Не разум. Функция.

**Как было до трансформера — T9.**
T9 в старых телефонах видел одно последнее слово и предлагал следующее. Контекст — одно слово. Написал «кошка» — предложил «лежит». Всё.

**Как стало с трансформером.**
«Собака лает на курицу» → T9 видит «курицу» и добавляет случайное наречие. Трансформер видит всё предложение — понимает что «лает» + «собака» + «курица» это агрессия — и добавляет «громко».

До: читал слово за словом, к концу забывал начало.
После: видит весь текст сразу, связывает слова по смыслу.

**Механизм внимания (Attention, 2018).** Каждый токен «разговаривает» с каждым другим токеном и решает — кто из них важен для его смысла. Именно поэтому когда модель долго думает — это буквально происходит: токены сверяются между собой попарно. Статья «Attention Is All You Need» — Google, 2018.

Pre-trained значит что он много всего знает заранее — до того как ты с ним поговорил. Важно, но без трансформера был бы просто умный поиск.

## 1.3 Токены

Модели работают не со словами — с **токенами** (кусочками текста).

Токен — минимальная единица языка для AI. Может быть меньше слова, больше слова, или ровно слово. Правила нет.

```
«Привет мир»  → 3 токена: «При», «вет», «мир»
«Hello world» → 2 токена: «Hello», «world»
```

**Почему важно:**
- Стоимость API = количество токенов
- Контекстное окно измеряется в токенах
- Русский в ~1.5× дороже английского
- 1000 токенов ≈ 750 слов

**Пиши промпты на английском.** На русском одно слово — больше токенов. Плюс данных для обучения на английском было в разы больше — модель лучше понимает контекст, тоньше чувствует ситуацию. На русском знает грамматику, но нюансов меньше. Практика: промпт на английском, интерфейс и ответы на русском — норм.

`/usage` в Claude Code — смотришь сколько токенов использовал.

## 1.4 Эмбеддинги

Все слова и токены — это числа. Просто числа в огромной матрице.

«Птица» и «корова» — два числа. Они стоят близко в этом пространстве, потому что часто встречаются в похожих контекстах (луг, природа, животные). «Птица» и «финансовый кризис» — далеко.

Это называется **эмбеддинги** — каждый токен занимает своё место в многомерном пространстве. Место определяет смысл.

Когда модель генерирует ответ, она буквально смотрит кто стоит рядом с твоим токеном в этом пространстве — и предлагает соседей.

Это не метафора. Это числа. Векторы. Направления.

## 1.5 Температура

При предсказании токена модель видит список вероятностей. Например: синий 40%, зелёный 30%, красный 15%, остальные по чуть-чуть.

Температура управляет насколько строго она им следует:

| Температура | Поведение | Когда |
|-------------|-----------|-------|
| 0 | Детерминированно, самый вероятный | Код, числа, факты |
| 0.7 | Баланс | Большинство задач |
| 1.5+ | Хаотично, случайный выбор | Только брейншторм |

**Разные модели — разная температура по умолчанию:**
- **Perplexity** — самая низкая. Скучно, зато точно. Хорош для поиска фактов.
- **Claude** — середина. Баланс между точностью и живостью.
- **ChatGPT** — выше. Самый «творческий», иногда несёт дичь.

Нужна сумасшедшая идея — иди в ChatGPT. Хочешь проверить логику — Perplexity. Строишь продукт — Claude.

Температуру можно задать явно в промпте: «будь максимально конкретным, не добавляй лишнего».

Одинаковый промпт → разные ответы — норма, не баг.

`/model` в Claude Code — переключить модель.

## 1.7 Контекстное окно

Контекстное окно — сколько информации модель удерживает в голове одновременно. Измеряется в токенах.

Аналог — буфер обмена. Только вместо одной скопированной штуки — весь ваш разговор.

**Чем больше контекст — тем лучше?** Не всегда.

Нужно узнать слово «кровать» по-французски — лучше спросить у студента-лингвиста, а не у человека со 153 учебниками разных языков. Модель с огромным контекстом будет долго думать, учтёт историю разговора о расставании, и в конце спросит «как дела у мужа».

**Правило:** разные задачи — разные чаты. Не валишь всё в одно место. Начал новую тему — новый чат.

Когда Claude говорит «давай сожмём наш разговор» — оптимизирует контекст. Это нормально.

## 1.8 Системный промпт

У каждой модели есть системный промпт — скрытые инструкции, которые идут до твоего сообщения.

В claude.ai это написал Anthropic: не пиши про насилие, будь полезным, не помогай с незаконным. Ты его не видишь — он просто есть.

Когда строишь свой продукт с API — ты сам пишешь системный промпт. Ты решаешь что боту можно, что нельзя, каким голосом говорит, что знает.

```
Ты [имя бота]. [Как говоришь]. [Что делаешь].
[Любые правила].
```

Системный промпт = личность бота. Без него — просто базовая модель.

## 1.9 Почему AI врёт уверенно

Модель оптимизирована на **правдоподобность**, а не на правдивость.

Она предсказывает что статистически вероятно после твоего вопроса. Не то что является правдой. Поэтому звучит уверенно даже когда неправа.

**Она не знает что не знает.** Нет внутреннего сигнала «здесь белое пятно». Если спросишь «ты уверен?» — она вполне может снова соврать, уже подтвердив ошибку. Это можно делать по кругу.

**Числа особенно опасны.** Модель не хранит числа точно — она их реконструирует из контекста. Простой калькулятор надёжнее.

**Правила:**
- Хочешь точные числа → проси Python-скрипт → запускай скрипт.
- Хочешь проверить факт → ищи в надёжных источниках.
- Не спрашивай «ты уверен» — это не метод верификации.

## 1.10 Три способа хранить данные

**localStorage** — в браузере, только ты, только здесь.
Закрыл вкладку → открыл → есть. Другой браузер → пусто. Другой человек → никогда.

Сделала to-do лист, добавила задачи, нажала F5 — есть. Зашла с телефона — пусто.

**Supabase** — в облаке, любой с доступом, любое устройство.
Открыл на телефоне → есть. Другой человек → видит. Именно это нужно для настоящего продукта.

**Claude API** — твой продукт разговаривает с внешним сервисом.
Кнопка «улучши текст» → Claude думает → ответ появляется. AI живёт внутри твоего продукта.

Строим все три сегодня.

## 1.11 Что такое API и Claude API

API — способ для программ общаться по сети.

```
Твой продукт → запрос → Claude API → ответ → Твой продукт
```

**Claude API в продукте** = AI живёт внутри. Не ты идёшь в claude.ai — твой продукт сам спрашивает и показывает ответ пользователю.

**OpenRouter** — один ключ, все модели (Claude, Gemini, GPT). Регистрируемся на openrouter.ai.

## 1.12 GitHub, Vercel, Supabase, Cursor

**GitHub** — хранилище кода. История изменений. Совместная работа.

**Vercel** — деплой за 30 секунд. Подключаешь к GitHub → каждый `git push` = автодеплой.

**Supabase** — база данных в облаке. Бесплатно. Без сервера.

**Cursor** — просто редактор кода с AI внутри. Ничего сверхъестественного. VS Code с кнопкой «спроси модель». Всё что он делает — редактирует код. Остальное — та же модель.

Связка: Строитель пишет код → пушит в GitHub → Vercel автодеплоит → Публикатор видит результат на живой ссылке.

## 1.1 Что такое языковая модель на самом деле

Языковая модель — это **функция предсказания следующего слова**.

Не база знаний. Не поисковик. Не разум. Функция которая смотрит на всё что было написано до — и предсказывает что идёт дальше.

Представь предложение: «Москва — столица...». Что будет следующим словом? «России» — с очень высокой вероятностью. «Мечты» — с низкой. «Холодильника» — почти ноль.

Модель обучена на триллионах текстов. Она научилась что обычно идёт после чего. И когда ты пишешь промпт — она предсказывает наиболее вероятное продолжение.

**Это объясняет все странности:**
- Почему модель уверенно врёт — она генерирует «правдоподобный» текст, а не «истинный»
- Почему одинаковый промпт даёт разные ответы — предсказание вероятностное
- Почему модель «соглашается» с тобой — следующее слово после «ты прав» статистически более вероятно чем «ты неправ»

## 1.2 Трансформеры и механизм внимания

Современные модели (GPT, Claude, Gemini) построены на архитектуре **трансформер**.

Главная идея трансформера — **механизм внимания (attention)**. Он позволяет модели при предсказании следующего слова смотреть на все предыдущие слова одновременно и решать — на какие из них обратить внимание.

**Пример:** «Врач сказал пациенту что **он** болен».
Кто «он» — врач или пациент? Механизм внимания позволяет модели установить связь между «он» и правильным существительным, даже если они далеко друг от друга в тексте.

**Что это значит практически:**
- Модель понимает контекст и зависимости между словами
- Чем длиннее текст — тем больше вычислений нужно
- У внимания есть предел — контекстное окно

## 1.3 Токены: как модель «видит» текст

Модели работают не со словами, а с **токенами** — кусочками текста.

Один токен ≈ 4 символа ≈ ¾ слова в английском. В русском — обычно больше символов на токен.

```
«Hello world» → [«Hello», « world»] → 2 токена

«Привет мир» → [«При», «вет», « мир»] → 3 токена (больше!)

«вайбкодинг» → [«вай», «б», «кодинг»] → 3 токена
```

**Почему важно:**
- Стоимость API считается в токенах (входящие + исходящие)
- Контекстное окно измеряется в токенах
- Редкие слова, имена, термины = больше токенов = дороже
- Код обычно токенизируется эффективнее чем текст

**Практическое правило:** 1000 токенов ≈ 750 слов ≈ 1.5 страницы текста.

**Стоимость примерно:**
- Claude Sonnet: ~$3 за 1 млн токенов на входе, ~$15 за выход
- Claude Opus: ~$15 за вход, ~$75 за выход
- Gemini Flash: $0.07–0.30 за 1 млн токенов

## 1.4 Температура и вероятностная природа генерации

Когда модель предсказывает следующий токен — она видит список вероятностей:

```
«Москва — столица»:
→ «России»: 94%
→ «нашей страны»: 3%
→ «мира»: 1%
→ всё остальное: 2%
```

**Температура** (temperature) — параметр который управляет насколько модель следует этим вероятностям.

| Температура | Поведение | Когда использовать |
|-------------|-----------|-------------------|
| 0 | Всегда выбирает наиболее вероятный токен. Детерминированно. | Код, факты, точные ответы |
| 0.5–0.7 | Иногда выбирает менее вероятные варианты. Баланс. | Большинство задач |
| 1.0+ | Выбирает случайно из широкого спектра. Творчески. | Brainstorm, креатив |
| 2.0+ | Хаос. Текст разваливается. | Никогда |

**Другие параметры:**
- **top-p (nucleus sampling):** берём только токены которые в сумме дают p% вероятности. top-p=0.9 = берём топ токены которые дают 90% совокупной вероятности.
- **top-k:** берём только k наиболее вероятных токенов. top-k=40 = только 40 вариантов на каждом шаге.

**Что это значит практически:**
- При temperature=0 одинаковый промпт даёт одинаковый ответ
- При temperature>0 каждый раз ответ немного отличается — это норма, не баг
- Для кода хочется низкую температуру
- Claude Code использует что-то близкое к 0–0.5 для надёжности

## 1.5 Контекстное окно: рабочий стол AI

**Контекстное окно** — это всё что модель «видит» в один момент времени. Системный промпт + история разговора + твой запрос + файлы которые ты дал.

Если текст не попал в контекстное окно — модель о нём не знает. Вообще. Никак.

**Размеры контекстов (2025):**
- Claude: 200 000 токенов (~150 000 слов — вся «Война и Мир»)
- GPT-4o: 128 000 токенов
- Gemini 1.5 Pro: 1 000 000 токенов (экспериментально)

**Проблема:** большой контекст ≠ хорошая работа с ним.

Исследования показывают: большинство моделей начинают хуже справляться с информацией из середины длинного контекста («lost in the middle» problem). Модель лучше «помнит» начало и конец. Середина теряется.

**Что происходит когда контекст переполняется:**

Claude Code автоматически запускает `compact` — сжимает историю разговора. Теряет детали. Может «забыть» инструкции которые давал 10 минут назад.

Признаки что контекст переполнен:
- Модель игнорирует инструкции из начала разговора
- Противоречит сама себе
- Начинает делать то что ты явно запрещал

**Что делать:**
- `/context` в Claude Code — смотришь процент заполнения
- При 70%+ → новый чат с нужным контекстом
- Разбивай большие задачи на маленькие, каждая в своём чате

## 1.6 Почему модели галлюцинируют

Галлюцинация — это когда модель генерирует правдоподобный, уверенно звучащий текст, который не соответствует реальности.

**Причина:** модель оптимизирована на «следующий токен похожий на реальный текст», а не на «следующий токен который является правдой».

**Типичные паттерны галлюцинаций:**
- Несуществующие библиотеки и функции (выглядят как настоящие)
- Неправильные числа и статистика (звучат правдоподобно)
- Выдуманные цитаты и источники (форматированы как настоящие)
- Несуществующие API-эндпоинты (имена реалистичны)

**Почему модель не говорит «я не знаю»:**
Её обучали на текстах где авторы уверены в том что пишут. Неуверенность статистически редка. Поэтому модель уверена почти всегда — даже когда не права.

**Как бороться:**
1. Попроси модель указывать источники — она укажет настоящие если они есть
2. Для точных данных — используй Python-скрипты (детерминированы)
3. Проверяй код который запускается, а не только код который выглядит правдоподобно
4. Попроси покритиковать свой же ответ: «Что в этом коде может быть неправильным?»

## 1.7 Три закона вайбкодинга

**Закон 1: Мусор на входе → мусор на выходе**

Модель предсказывает продолжение твоего промпта. Если промпт размытый — продолжение тоже будет размытым.

```
❌ «Сделай кнопку»

✅ «Создай кнопку с тремя состояниями: обычное, загрузка (спиннер), 
   ошибка (красный цвет). Кнопка должна быть недоступна во время загрузки.»
```

**Закон 2: Доверяй, но проверяй**

Модель всегда звучит уверенно. Это не признак правоты. Всегда тестируй результат.

**Закон 3: Итерируй, не регенерируй**

Когда код неправильный — скажи конкретно что не так и попроси исправить. Не начинай заново. Контекст разговора помогает модели понять что именно ты хочешь.

## 1.8 Вайбкодинг: что это такое

**Вайбкодинг** — ты описываешь что хочешь словами, AI пишет код.

Не учишь синтаксис. Не гуглишь документацию. Не отлаживаешь вручную часами. Говоришь что нужно — получаешь работающий результат.

**FAAFO — пять измерений ценности:**

| | Измерение | Суть |
|--|-----------|------|
| **F** | Fast | Задачи которые занимали дни — занимают часы |
| **A** | Ambitious | Проекты которые казались невозможными — реальны |
| **A** | Autonomous | Делаешь в одиночку то что раньше требовало команды |
| **F** | Fun | Фокус на создание, а не на синтаксис |
| **O** | Optionality | Пробуешь несколько подходов параллельно |

**Ты шеф-повар, не повар:**
Режиссёр не держит камеру, но решает каждый кадр. Ты — режиссёр. AI — оператор. Делегирование реализации — не делегирование ответственности.

## 1.9 Чем Claude Code отличается от Claude в браузере

| | Claude в браузере | Claude Code |
|--|---|---|
| Файлы | Не видит | Читает и создаёт |
| Код | Показывает в чате | Запускает на твоём компьютере |
| Деплой | Не умеет | Деплоит на Vercel |
| Браузер | Не открывает | Ходит по сайтам |
| Инструменты | Нет | MCP, скрипты, терминал |

**Основные команды Claude Code:**
```
/model    → переключить модель (Opus / Sonnet)
/usage    → сколько лимита использовал
/context  → % заполнения контекстного окна
Escape    → остановить если думает слишком долго
```

## 1.10 PRD: описание продукта как инструкция для AI

**PRD** (Product Requirements Document) — текстовый файл где написано что ты хочешь построить.

**Зачем:** без PRD AI строит наугад. С PRD — точно по твоим требованиям.

**Пять частей хорошего промпта:**

| Часть | Вопрос | Пример |
|-------|--------|--------|
| Контекст | Что за ситуация? | Главред медиа о технологиях |
| Задача | Что хочешь? | Создать инструмент контент-плана |
| Требования | Какие ограничения? | Работает оффлайн, один пользователь |
| Формат | Как должно выглядеть? | HTML + CSS + JS, без фреймворков |
| Примеры | Какой паттерн следовать? | Как Notion, но проще |

**Прием «творческая свобода»:**
Добавь в конец промпта:
```
Ты можешь добавить любую функциональность которую считаешь 
необходимой или полезной. Используй своё суждение.
```
По умолчанию модели минимизируют объём. Эта фраза явно разрешает добавлять полезные вещи которые ты забыл указать.

## 1.1 Что такое языковая модель

Языковая модель — это **функция предсказания следующего слова**.

Не база знаний. Не поисковик. Не разум. Функция которая смотрит на всё что было написано до — и предсказывает что идёт дальше.

Представь предложение: «Москва — столица...». Следующее слово? «России» — с очень высокой вероятностью. «Мечты» — с низкой. «Холодильника» — почти ноль.

Модель обучена на триллионах текстов. Она выучила что обычно идёт после чего. Когда ты пишешь промпт — она предсказывает наиболее вероятное продолжение.

**Это объясняет все странности:**
- Почему модель уверенно врёт — она генерирует «правдоподобный» текст, а не «истинный»
- Почему одинаковый промпт даёт разные ответы — предсказание вероятностное
- Почему модель «соглашается» с тобой — «ты прав» статистически более вероятно чем «ты неправ»

## 1.2 Трансформеры и механизм внимания

Современные модели (GPT, Claude, Gemini) построены на архитектуре **трансформер**.

Главная идея — **механизм внимания (attention)**. При предсказании следующего слова модель смотрит на все предыдущие одновременно и решает — на какие обратить внимание.

**Пример:** «Врач сказал пациенту что **он** болен».
Кто «он»? Механизм внимания позволяет установить связь между «он» и правильным существительным, даже если они далеко.

**Практически это значит:**
- Модель понимает зависимости между словами в длинном тексте
- Чем длиннее текст — тем больше вычислений
- У этого механизма есть предел — контекстное окно

## 1.3 Токены: как модель видит текст

Модели работают не со словами, а с **токенами** — кусочками текста.

Один токен ≈ 4 символа. В русском обычно больше символов на токен чем в английском.

```
«Hello world»  → 2 токена
«Привет мир»   → 3 токена (больше!)
«вайбкодинг»   → 3-4 токена
```

**Почему важно:**
- Стоимость API = количество токенов (входящие + исходящие)
- Контекстное окно измеряется в токенах
- 1000 токенов ≈ 750 слов ≈ 1.5 страницы

**Стоимость примерно:**
- Claude Sonnet: ~$3 за 1 млн входящих, ~$15 за исходящие
- Gemini Flash: $0.07–0.30 за 1 млн токенов
- Claude Pro ($20/мес) = лимит в Claude Code

## 1.4 Температура и случайность генерации

При предсказании следующего токена модель видит список вероятностей:

```
«Москва — столица»:
→ «России»:       94%
→ «нашей страны»: 3%
→ «мира»:         1%
→ всё остальное:  2%
```

**Температура** управляет насколько модель следует этим вероятностям:

| Температура | Поведение | Когда |
|-------------|-----------|-------|
| 0 | Всегда наиболее вероятный. Детерминировано. | Код, факты |
| 0.5–0.7 | Иногда менее вероятные варианты | Большинство задач |
| 1.0+ | Широкий разброс. Творчески. | Брейншторм |

**Практически:** при temperature=0 одинаковый промпт = одинаковый ответ. При temperature>0 каждый раз чуть отличается — это норма, не баг.

## 1.5 Контекстное окно: рабочий стол AI

**Контекстное окно** — всё что модель «видит» прямо сейчас. Системный промпт + история разговора + твой запрос + файлы.

Не попало в контекст — модель не знает. Вообще.

**Размеры:**
- Claude: 200 000 токенов (~150 000 слов)
- GPT-4o: 128 000 токенов

**Важно:** большой контекст ≠ хорошая работа с ним. Исследования показывают — модели хуже справляются с информацией из середины длинного контекста.

**Признаки что контекст переполнен:**
- Модель игнорирует инструкции из начала разговора
- Противоречит сама себе
- Делает то что ты явно запрещал

**Что делать:** `/context` в Claude Code — видишь процент заполнения. При 70%+ → новый чат.

## 1.6 Почему модели галлюцинируют

Галлюцинация — правдоподобный уверенно звучащий текст который не соответствует реальности.

**Причина:** модель оптимизирована на «следующий токен похожий на реальный текст», а не на «следующий токен который является правдой».

**Типичные паттерны:**
- Несуществующие функции и библиотеки (выглядят как настоящие)
- Неправильные числа и статистика (звучат правдоподобно)
- Выдуманные API-эндпоинты (имена реалистичны)

**Как бороться:**
1. Запускай код — не просто смотри что он правдоподобен
2. Попроси покритиковать свой же ответ
3. Для точных данных — Python-скрипты (об этом в следующей главе)

## 1.7 Три закона вайбкодинга

**Закон 1: Мусор на входе → мусор на выходе**
```
❌ «Сделай кнопку»
✅ «Создай кнопку с тремя состояниями: обычное, загрузка (спиннер), 
   ошибка (красный). Недоступна во время загрузки.»
```

**Закон 2: Доверяй, но проверяй**
Модель всегда звучит уверенно. Это не признак правоты. Тестируй результат.

**Закон 3: Итерируй, не регенерируй**
Код неправильный → скажи конкретно что не так → попроси исправить. Не начинай заново.

## 1.8 Чем Claude Code отличается от Claude в браузере

| | Claude в браузере | Claude Code |
|--|---|---|
| Файлы | Не видит | Читает и создаёт |
| Код | Показывает в чате | Запускает |
| Деплой | Не умеет | Деплоит на Vercel |
| Браузер | Нет | Ходит по сайтам |

**Команды которые пригодятся сегодня:**
```
/model    → переключить модель
/usage    → сколько лимита использовал
/context  → % заполнения окна
Escape    → остановить
```

## 1.9 PRD: описание продукта как инструкция для AI

**PRD** — текстовый файл с тем что ты хочешь построить.

Без PRD AI строит наугад. С PRD — точно по твоим требованиям.

**Пять частей хорошего промпта:**

| Часть | Вопрос | Пример |
|-------|--------|--------|
| Контекст | Что за ситуация? | Главред медиа о технологиях |
| Задача | Что хочешь? | Инструмент для контент-плана |
| Требования | Какие ограничения? | Один пользователь, работает офлайн |
| Формат | Как выглядит? | HTML + CSS + JS, без фреймворков |
| Примеры | Какой паттерн? | Как Notion но проще |

**Приём «творческая свобода»:** добавь в конец:
```
Ты можешь добавить любую функциональность которую считаешь нужной.
```
По умолчанию модели минимизируют объём. Эта фраза разрешает добавлять полезное.

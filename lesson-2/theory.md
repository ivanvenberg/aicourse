## 1.1 Что такое нейросеть

Нейросеть — машина для последовательного преобразования данных через слои:

- **Слой 1** → простые признаки (края, линии)
- **Слой 2** → сложнее (формы, части)
- **Слой 3** → понятия (ухо, хвост)
- **Выходной** → ответ (кошка 97%)

Никто не говорит слоям что искать — они находят это сами в процессе обучения. Обучение = подбор миллиардов весов пока ошибка не станет минимальной.

---

## 1.2 Трансформер и языковая модель

Языковая модель — это **функция предсказания следующего токена**.

Не база знаний. Не разум. Функция.

**Трансформер** (2017, Google, статья «Attention Is All You Need») — архитектура на которой построены Claude, GPT, Gemini.

Главная идея — **механизм внимания**: каждый токен смотрит на все остальные и решает кто из них важен для его смысла.

До этого: читал слово за словом, к концу забывал начало.
После: видит весь текст сразу, связывает слова по смыслу.

---

## 1.3 Токены

Модели работают не со словами — с **токенами** (кусочками текста).

```
«Привет мир»  → 3 токена
«Hello world» → 2 токена
```

**Почему важно:**
- Стоимость API = количество токенов
- Контекстное окно измеряется в токенах
- Русский в 1.5× дороже английского
- 1000 токенов ≈ 750 слов

`/usage` в Claude Code — смотришь сколько использовал.

---

## 1.4 Температура

При предсказании токена модель видит список вероятностей.
Температура управляет насколько строго она им следует:

| Температура | Поведение | Когда |
|-------------|-----------|-------|
| 0 | Детерминированно | Код, числа |
| 0.7 | Баланс | Большинство задач |
| 1.5+ | Хаотично | Только брейншторм |

Одинаковый промпт → разные ответы — норма, не баг.

## 1.5 Как выбрать модель

| Задача | Модель |
|--------|--------|
| Бот, простые вопросы | Gemini Flash ($0.07/млн) |
| Текст, анализ, код | Claude Sonnet ($3/$15 млн) |
| Сложные решения | Claude Opus ($15/$75 млн) |

**Правило:** Sonnet по умолчанию. Opus только если Sonnet не справился.

`/model` в Claude Code — переключить модель.

## 1.6 Почему AI врёт уверенно

Модель оптимизирована на **правдоподобность**, а не на правдивость.

Она предсказывает что статистически вероятно после твоего вопроса — не то что является правдой. Поэтому звучит уверенно даже когда неправа.

**Правило:** хочешь точные числа → проси Python-скрипт → запускай скрипт.

## 1.7 Три способа хранить данные

**localStorage** — в браузере, только ты, только здесь.
Закрыл вкладку → открыл → есть. Другой браузер → пусто.

**Supabase** — в облаке, любой с доступом, любое устройство.
Открыл на телефоне → есть. Другой человек → видит.

**API** — твой продукт разговаривает с внешним сервисом.
Claude API: кнопка «улучши текст» → Claude думает → ответ появляется.

Строим все три сегодня.

---

## 1.8 Что такое API

API — способ для программ общаться по сети.

```
Твой продукт → запрос → Claude API → ответ → Твой продукт
```

**Claude API в продукте** = AI живёт внутри. Не ты идёшь в claude.ai — твой продукт сам спрашивает и показывает ответ пользователю.

**OpenRouter** — один ключ, все модели (Claude, Gemini, GPT). Регистрируемся на openrouter.ai.

## 1.9 GitHub, Vercel, Supabase

**GitHub** — хранилище кода. История изменений. Совместная работа.

**Vercel** — деплой за 30 секунд. Подключаешь к GitHub → каждый `git push` = автодеплой.

**Supabase** — база данных в облаке. Бесплатно. Без сервера.

Связка: Строитель пишет код → пушит в GitHub → Vercel автодеплоит → Публикатор видит результат на живой ссылке.

---

## 1.10 Роли в паре

**Строитель** — пишет PRD, строит продукт, пушит в GitHub.

**Публикатор** — создаёт GitHub репо, подключает Vercel, настраивает Supabase, деплоит.

В домашке роли меняются — каждый пробует обе стороны.

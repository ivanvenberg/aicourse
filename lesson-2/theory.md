## Что такое терминал

Терминал — это текстовое окошко где можно давать команды компьютеру напрямую. Без кнопок и меню.

Ты уже умеешь пользоваться терминалом — когда делал `git pull` на прошлом уроке.

На Mac: приложение **Terminal** (или iTerm2)
На Windows: **Git Bash** или **PowerShell**

Не бойся терминала. Это просто другой способ разговаривать с компьютером.

## Чем Claude Code отличается от Claude в браузере

**Claude в браузере (claude.ai):**
- Отвечает текстом
- Не может трогать файлы на твоём компьютере
- Не может запускать код
- Не знает что лежит в твоих папках

**Claude Code (в терминале):**
- Читает и создаёт файлы на твоём компьютере
- Запускает код
- Устанавливает программы
- Открывает браузер и ходит по сайтам
- Деплоит на Vercel
- Видит весь твой проект

Это как разница между советчиком по телефону и человеком который пришёл к тебе домой и сам всё сделал.

## Что такое вайбкодинг

Вайбкодинг — это когда ты описываешь что хочешь словами, а AI пишет код.

Ты не учишь синтаксис. Ты не отлаживаешь ошибки вручную. Ты говоришь что нужно — и получаешь работающий результат.

**Главный навык вайбкодинга:**
> «Я не знаю в чём проблема. Вот что должно происходить. Почини.»

Именно это умение — описывать результат, а не процесс — делает тебя сильным вайбкодером.

## Что такое PRD

PRD (Product Requirements Document) — документ с описанием продукта.

Для нас это просто текстовый файл где написано:
- Для кого этот продукт
- Какую проблему решает
- Что должно работать
- Как это должно выглядеть

**Зачем нужен PRD?**

Без PRD Claude Code строит наугад. С PRD — точно по твоим требованиям.

Чем точнее ты описал продукт → тем точнее он его построит.

**Как писать PRD:**

Не нужно знать технический язык. Пиши как объясняешь другу.

Пример для главреда:
```
Я главред медиа про технологии.
Мне нужен инструмент который помогает планировать контент.

Пользователь: я, один человек.

Что должно работать:
- Добавить тему с описанием и дедлайном
- Видеть все темы в виде списка
- Отмечать тему как готовую
- Фильтровать по статусу

Дизайн: минималистичный, тёмный фон, без лишнего.
```

Это и есть PRD. Ничего сложного.

## Как описывать задачи Claude Code

Три принципа:

**1. Описывай результат, не процесс**
❌ «Напиши функцию которая обрабатывает клик»
✅ «Когда пользователь нажимает кнопку, добавь задачу в список»

**2. Давай контекст**
❌ «Сделай красиво»
✅ «Я дизайнер, моя аудитория — творческие студии. Сделай лаконичный дизайн без лишних деталей, шрифт крупный, много воздуха»

**3. Говори что не работает**
❌ «Сломалось»
✅ «Я нажимаю кнопку "Добавить", ничего не происходит. Консоль (F12) показывает: Cannot read property of null»

## Что такое Vercel и зачем деплоить

Vercel — сервис который берёт твои файлы и публикует их в интернете.

До деплоя: сайт существует только на твоём компьютере.
После деплоя: у сайта есть ссылка которую можно отправить кому угодно.

Деплой занимает 30 секунд. Claude Code делает это сам.

## Как установить Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

Если npm не установлен — сначала установи Node.js: [nodejs.org](https://nodejs.org) → кнопка LTS.

После установки:
```bash
claude
```

Залогинься через claude.ai аккаунт. Всё готово.

**Полезные команды:**
- `/model` — переключить модель (Opus / Sonnet)
- `/usage` — посмотреть сколько лимита использовал
- `/context` — сколько занято контекстное окно
- `Escape` — остановить если думает слишком долго

## Частые страхи — и правда

**«Я не понимаю что он делает»**
Это нормально. Спроси: «Объясни что ты только что сделал простыми словами».

**«Он что-то сломал»**
Скажи: «Ты сломал то что работало. Верни как было».

**«Я не знаю что написать»**
Начни с: «Задай мне вопросы, из ответов на которые ты поймёшь что я хочу сделать».

**«Вдруг я напишу не то»**
Claude Code не запустит ничего опасного без твоего разрешения. Он всегда спрашивает прежде чем что-то изменить.

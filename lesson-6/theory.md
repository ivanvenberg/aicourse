## 5.1 Что значит AI-native

Не «знаю ChatGPT». А:

```
✓ Формулирую задачи под агента (образ целиком, не пиксели)
✓ Декомпозирую — маленькие шаги с проверкой
✓ Знаю, когда скрипт, когда RAG, когда Perplexity/Parallel
✓ Держу секреты в .env, данные в БД
✓ Ревьюю diff — принял код = мой код
✓ Измеряю: токены, время, качество (golden tests)
✓ Собираю скиллы и MCP под свою профессию
```

## 5.2 Git — страховка от агента

AI **удалит** файл без предупреждения. Git — машина времени.

```bash
git add .
git commit -m "chore: перед рефактором auth"
# ... агент работает ...
# сломалось:
git checkout -- .
# или
git reset --hard HEAD
```

### Семантические коммиты

```
feat: добавить RAG по knowledge.md
fix: бот не отвечал на пустое сообщение
refactor: вынести API в api.js
docs: обновить README
chore: обновить зависимости
```

История читается как дневник.

### Ветки для экспериментов

```bash
git checkout -b experiment/new-ui
# агент крутит...
# понравилось → merge
# нет → checkout main && branch -D experiment/new-ui
```

## 5.3 Code review AI-кода

Ты **не обязан** читать каждую строку. Обязан **понимать**, что принимаешь.

**Четыре приёма:**

1. **Объясни как пятилетнему**
```
Объясни этот файл простыми словами. Что может сломаться?
```

2. **Злой ревьюер**
```
Притворись строгим ревьюером. Найди баги и дыры безопасности.
```

3. **Чеклист**
```
□ .env не в коде
□ ошибки обработаны
□ пустой ввод
□ нет debug-логов
```

4. **Diff перед accept**
В Cursor смотри, что **удалено**. Много красного — стоп, спроси зачем.

## 5.4 Технический долг

Вайбкодинг **быстрый** → долг копится **быстро**:
- дубли функций
- `interim_result_5`
- закомментированный код
- «временно» без TODO

**После каждой крупной фичи:**
```
Найди дубли, мусор, функции > 30 строк. Упрости, не ломая поведение.
```

На дедлайне — только критичное. Остальное в `TECH-DEBT.md`.

---

## 5.5 Когда перестать доверять агенту

| Сигнал | Действие |
|--------|----------|
| Не понимаешь diff | Попроси объяснить или упрости задачу |
| Одна ошибка 5 раз | Новый чат, другая формулировка |
| Ломается несвязанное | Нарисуй схему на бумаге, потом агенту |
| Auth, деньги, PII | Каждую строку сам |

**Human-in-the-loop** — AI черновик, **ты** финальное решение на важном.

## 5.6 Стоимость AI-продукта

```
Счёт ≈ (входные токены + выходные токены) × цена модели × число пользователей
```

**Как экономить:**
- Flash для рутины, Sonnet для сложного
- Короткий системный промпт
- RAG top-3, не top-20
- Кэш частых ответов
- Лимит запросов на пользователя

**Пример:** бот 1000 сообщений/день × 500 токенов × Flash — посчитай в Excel **до** запуска.

## 5.7 Полная карта стека AI-native

```
┌─────────────────────────────────────────────────────────┐
│  ТЫ: задача, ревью, этика, продуктовые решения          │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  РЕДАКТОР: Cursor + агент (файлы, терминал, git)       │
│  Скиллы · CLAUDE.md · MEMORY.md                        │
└──────────────────────────┬──────────────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
   ┌─────────┐       ┌───────────┐      ┌────────────┐
   │  MCP    │       │ Субагенты │      │  Скрипты   │
   │ Notion  │       │ параллель │      │  Python    │
   │ Sheets  │       │ задачи    │      │  точные    │
   │ Figma   │       └───────────┘      │  цифры     │
   └─────────┘                          └────────────┘
        │
        ▼
┌───────────────────────────────────────────────────────┐
│  ВНЕШНИЕ API                                          │
│  OpenRouter (модели) · Perplexity (факты+ссылки)      │
│  Parallel (поиск/обогащение) · Image API              │
└───────────────────────────┬───────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
   ┌─────────┐        ┌──────────┐       ┌───────────┐
   │ Supabase│        │ Telegram │       │  Vercel   │
   │ БД+RAG  │        │  бот     │       │  деплой   │
   └─────────┘        └──────────┘       └───────────┘
```

**Уроки 1–3:** низ (модель, промпты, БД, скрипты).  
**Урок 4:** середина (агент, MCP, поиск).  
**Урок 5:** API-продукт (RAG, бот, безопасность).  
**Урок 6:** ты сверху (git, ревью, деньги, процесс).

## 5.8 Что автоматизируется, что нет

| Автоматизируется | Остаётся за тобой |
|------------------|-------------------|
| Шаблонный код | Зачем это нужно людям |
| Тесты, доки | Верно ли для **твоего** случая |
| Рутина в Sheets/Notion | Этика, ответственность |
| Первый черновик | Финал для клиента |
| Поиск по докам (RAG) | Какие доки класть в базу |

**Ошибся бот — твой бот.** AI не несёт ответственности.

---

## 5.9 Деплой и домен (коротко)

```
git push → GitHub → Vercel → https://....vercel.app
```

**Домен:** купил → DNS-записи от Vercel → `https://твой.ру`

Продукт AI-native = **ссылка работает**, ключи на сервере в env vars (не в репо).

| Направление | Инструменты |
|-------------|-------------|
| Глубже агенты | MCP, больше скиллов, Parallel Task |

**Не гонись за всем.** Один стек, доведённый до автоматизации рутины — уже AI-native.

## 5.11 Чеклист «я AI-native» (самопроверка)

- [ ] Могу задеплоить продукт с AI внутри
- [ ] RAG или knowledge с «не знаю»
- [ ] Ключи в .env, .gitignore настроен
- [ ] 3+ скилла или MCP под работу
- [ ] Знаю цену своего API в месяц (хотя бы порядок)
- [ ] Golden tests на AI-ответы
- [ ] git commit перед большой правкой агента
- [ ] Использовал Perplexity или Parallel для фактов из сети
- [ ] Могу объяснить свой код простыми словами

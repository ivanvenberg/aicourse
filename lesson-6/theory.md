## 5.1 Git — страховка

AI удаляет файлы. `git commit` — точка возврата.

**Перед большой правкой агентом:**
```bash
git add .
git commit -m "chore: перед [что делаем]"
```

**Семантические коммиты:**
```
feat: платная фича в app
fix: пустое поле ломало форму
docs: обновил brief
```

Смотри `git diff` — много удалений → спроси зачем.

## 5.2 Code review AI-кода

Ты не писал строки — ты **принял** их. Значит, твоя ответственность.

**Промпты:**
```
Объясни код как непрограммисту.
```
```
Ты злой ревьюер. Что сломается у покупателя на демо?
```
```
Чеклист: ошибки, пустой ввод, ключи не в коде, дубли.
```

---

## 5.3 Техдолг

Вайбкодинг быстрый → мусор накапливается быстро: логи, `temp2`, закомментированный хлам.

После крупной фичи:
```
Убери debug, мёртвый код, дубли. Не ломай поведение.
```

На горящем демо — только критичное. Остальное — в план на месяц.

---

## 5.4 Когда взять руль

- Перестал понимать diff  
- Агент крутится на одной ошибке  
- Платежи, персональные данные, права доступа  

Тогда: упрости задачу, новый чат, или одну функцию руками с агентом «объясни построчно».

## 5.6 Питч 5 минут — не «я учился»

| Время | Блок |
|-------|------|
| 0:00 | **Кому** продаёшь |
| 0:30 | **Боль в деньгах** |
| 1:00 | **Живое демо** платной фичи |
| 3:00 | **Цена** и что входит |
| 3:30 | **Цитата** покупателя |
| 4:00 | **Шаг на месяц** — первый платящий / пилот |

Сломалось на демо — чини вслух. Честность > идеальный клик.

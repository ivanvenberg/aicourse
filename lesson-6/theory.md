## 6.2 Деплой: от localhost к реальному миру

**Vercel** — то что мы использовали. Идеально для фронтенда и небольших проектов.

**Railway** — лучше для Telegram-ботов и бэкенда с базой данных. Бот будет работать 24/7.

**Домен** — твой адрес в интернете. Купить можно на namecheap.com, ~$10/год.

**Как подключить домен к Vercel:**
```
Я купил домен [твой домен].
Помоги подключить его к моему проекту на Vercel.
Объясни какие DNS-записи нужно добавить.
```

**Финальная проверка перед деплоем:**
```
Перед деплоем проверь:
1. Нет ли API ключей в коде (должны быть в .env)
2. Работает ли авторизация если есть
3. Корректно ли обрабатываются ошибки
4. Работает ли на мобильном телефоне
Исправь всё найденное.
```

## 6.3 Почему важно показывать продукт

> *«Vibe coding создаёт ценность по пяти измерениям. Самое мощное из всех — Optionality (опциональность). Вместо того чтобы делать ставку на одно решение — попробуй несколько. Каждый раз когда показываешь продукт живым людям — ты узнаёшь что-то что не мог узнать иначе.»*

Самая частая ошибка вайбкодеров — сделать и спрятать.

**Почему показывать важно:**
- Реальные пользователи находят то что ты не видишь

**Правило пяти:**
Покажи продукт пяти живым людям. Не коллегам по курсу. Не маме. Пяти людям которые могли бы реально использовать продукт. Запиши что они сказали. Сделай хотя бы одно улучшение на основе их слов.

## 6.5 Тёмная сторона которую важно знать

**«Eldritch Horror Codebase»** — реальная история:
После многих часов вайбкодинга его код превратился в «один трёхтысячестрочный монстр без модульных границ — невозможно понять или изменить не сломав что-то ещё». Три дня потратил на рефакторинг.

**Что это означает для тебя:**
Периодически проси Claude Code «навести порядок»:

```
Посмотри на мой проект.
Найди где код стал запутанным или трудночитаемым.
Разбей на модули.
Удали неиспользуемый код и комментарии.
Сделай так чтобы следующему человеку было понятно что происходит.
```

**«AI — слизняк и мусорщик»:**
AI оставляет после себя:
- Debug statements которые заполняют консоль
- Неиспользуемые переменные с именами `interim_result5`
- Комментарии «// keeping this for now»
- Временные файлы разбросанные везде

Регулярно проси убраться:
```
Почисти код: удали все debug-логи, неиспользуемые переменные,
закомментированный код и временные файлы.
```

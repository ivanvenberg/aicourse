# Подготовка к демо — Урок 6

**Дедлайн:** до начала встречи  
**Это не домашка после — это чеклист перед выходом в зал**

---

## За 2 дня до демо

- [ ] Продукт по ссылке открывается с телефона
- [ ] Лендинг → app → платная фича за &lt; 2 минут
- [ ] `brief.md` актуален (цена, покупатель, фича)
- [ ] `feedback.md` — минимум одна цитата вне курса

---

## За 1 день

```bash
git add .
git commit -m "feat: финальная версия перед демо"
```

Попроси агента:

```
Строгий code reviewer. brief.md в корне.
Что сломается у покупателя на демо? Исправь критичное.
Ключи только в .env.
```

## За 1 час

Репетиция **вслух**, таймер 5:00:

| Сек | Что |
|-----|-----|
| 0–30 | Кому продаёшь |
| 30–60 | Боль в деньгах |
| 60–180 | Живое демо платной фичи |
| 180–210 | Цена |
| 210–240 | Цитата покупателя |
| 240–300 | Один шаг на месяц |

Запиши голосовуху себе или напарнику. Слушай — где воды? Вырежь.

---

## В день демо

- [ ] Ссылка в закладках
- [ ] Запасной план если Wi‑Fi умрёт (скринкаст 30 сек или локальный запуск)
- [ ] Одна фраза на месяц: «Сделаю ___ для ___ до [дата]»

---

## Минимум зачёта

Ссылка + `brief.md` + цитата «куплю» + питч 5 мин. Домен — бонус.

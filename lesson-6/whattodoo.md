# Урок 6. Продукт в мире

**Сегодня каждый деплоит продукт на домен и презентует его группе.**

---

## Подготовка до встречи

**1. Git-порядок**
```
Сделай коммит с сообщением:
feat: финальная версия перед демо-днём
```

Потом проверь историю:
```bash
git log --oneline -10
```
Видишь понятные сообщения? Если нет — перепиши:
```
Исправь последние 5 коммитов чтобы сообщения были семантическими:
feat/fix/refactor/docs + краткое описание
```

**2. Ревью своего кода**
```
Сыграй роль строгого code reviewer.
Найди в моём продукте:
- дублирующийся код
- непонятные имена переменных
- debug-логи которые забыли убрать
- что может сломаться в продакшне
Исправь.
```

**3. Безопасность**
```
Проверь что все API ключи в .env, а не в коде.
Проверь что нет уязвимостей которые нашли на прошлом уроке.
```

**4. Одно предложение**
Закончи: *«Мой продукт помогает [кому] делать [что] без [чего].»*

## Структура встречи

### Часть 1. Демо (60 мин)

Каждый выходит. **5 минут:**
1. Проблема (30 сек)
2. Живое демо (2 мин)
3. Результат (1 мин)
4. Что узнал (30 сек)
5. Что дальше (30 сек)

После каждого — 2 минуты фидбека от группы.

## 🔥 Упражнение: «Code review чужого продукта»

*Практика из главы 5: code review кода которого ты не писал.*

Меняемся продуктами в парах.

Каждый открывает Claude Code в папке продукта соседа и делает:

```
Ты — строгий code reviewer. Прочитай весь код в этом проекте.
Напиши отчёт:
1. Что сделано хорошо
2. Что потенциально опасно или может сломаться
3. Три конкретных улучшения которые ты бы сделал
Будь конкретным — называй файлы и строки.
```

Потом показываешь отчёт автору продукта. Автор слушает, задаёт вопросы.

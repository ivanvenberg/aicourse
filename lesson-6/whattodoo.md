## До встречи (домашка-подготовка)

### 1. Git-порядок

```bash
git add .
git commit -m "feat: финальная версия перед демо"
git log --oneline -5
```

Непонятные сообщения? Попроси агента переписать последние коммиты семантически.

### 2. Ревью

```
Строгий code reviewer. Проект из brief.md.
Найди: дубли, мусор, ключи не в .env, что сломается у покупателя.
Исправь критичное.
```

### 3. Проверь brief

- Цена актуальна?
- Цитата «куплю» на месте?
- Платная фича совпадает с тем, что покажешь?

### 4. Репетиция питча

Вслух 5 минут по таблице из `theory.md` §5.6. Таймер.

## Часть 1. Демо-питчи (60 мин)

**5 минут на человека:**

| Минута | Содержание |
|--------|------------|
| 0:00 | Кому продаёшь |
| 0:30 | Боль в деньгах |
| 1:00 | Живое демо **платной фичи** |
| 3:00 | Цена и что входит |
| 3:30 | Цитата покупателя |
| 4:00 | Один шаг на месяц |

**2 минуты фидбек** от группы — только как покупатель:
- За что заплатил бы?
- Что непонятно?
- Что убрать?

---

## 🔥 Упражнение: «Code review чужого продукта»

Пары. Открой репо соседа в Cursor:

```
Строгий code reviewer. Прочитай весь код.
1. Что хорошо
2. Что опасно для покупателя
3. Три конкретных улучшения (файл + строка)
```

5 минут — устно автору. Автор записывает один пункт «сделаю в течение недели».

---

## 🔥 Упражнение: «Аукцион цены»

После питча — зал **анонимно в чат** одной цифрой: сколько заплатил бы в месяц (0 = не купил бы).

Автор видит разброс. 30 сек — реакция: удивлён / ожидал / понял что менять в оффере.

Не стыдим за 0. Это данные.

## Часть 3. Закрытие (10 мин)

## 3.1 Главное правило работы с данными

Это самый важный инсайт всей главы. Запомни его:

> **Хочешь точные цифры → проси написать Python-скрипт → запускай скрипт → доверяй результату.**

**Почему нельзя просто спросить у AI:**

Языковая модель — это не калькулятор. Она **предсказывает слова**. Она не считает — она угадывает правдоподобный ответ. И делает это с полной уверенностью, даже когда ошибается.

Эксперимент который показывает это лучше всяких слов:

```
Все читают один файл reviews.json.
Все спрашивают AI: «Посчитай NPS».
Результаты:
- Аня: NPS = 36
- Дима: NPS = 24
- Катя: NPS = 41
- Саша: NPS = 29.7

Данные ОДНИ. AI ГАЛЛЮЦИНИРУЕТ у каждого по-своему.
```

Потом все просят AI написать Python-скрипт и запускают его:

```
Все получают: NPS = 29.7
Вариативность исчезает.
```

Скрипт детерминирован: один код + одни данные = один результат всегда.

## 3.2 Как устроено AI мышление

> *«AI похож на гения-самозванца: он всегда звучит убедительно, даже когда полностью неправ. Он объяснит тебе почему 2+2=5 таким образом, что ты почти поверишь. Это не баг — это фундаментальная особенность того как работают языковые модели.»*

AI обучен **предсказывать следующий токен** — следующее слово. Когда ты просишь посчитать — он предсказывает какое число выглядит правдоподобно в данном контексте. Не вычисляет. Предсказывает.

Поэтому:
- Для **текста, анализа, объяснений** — AI отлично подходит
- Для **точных числовых расчётов** — нужен скрипт

## 3.3 Что такое Python-скрипт и зачем его просить

Скрипт — небольшая программа которая делает одну конкретную задачу. Детерминированно. Всегда одинаково.

Тебе **не нужно знать Python**. Ты просишь Claude Code написать скрипт, Claude Code его запускает, ты видишь результат.

**Для каких задач нужны скрипты:**
- Считать NPS, средний чек, конверсии
- Анализировать воронку
- Обрабатывать CSV-файлы
- Любые точные числовые расчёты

**Примеры промптов:**

*Для CRM маркетолога:*
```
Напиши Python-скрипт который читает base.csv.
Посчитай: средний LTV, отток за последние 3 месяца,
долю активных пользователей.
Выведи результат понятно.
```

*Для СМО:*
```
Напиши скрипт который читает analytics.csv.
Посчитай ROAS по каждому каналу.
Найди три канала с лучшим показателем.
```

*Для главреда:*
```
Напиши скрипт который читает articles.csv
(колонки: views, time_on_page, bounce_rate).
Найди 5 материалов с лучшим вовлечением.
```

## 3.4 Что такое база данных

**Без базы данных:**
Пользователь добавил задачу → закрыл браузер → задача исчезла. Данные жили только в памяти браузера (localStorage).

**С базой данных:**
Данные хранятся в облаке → открыл на телефоне через неделю → всё на месте → другой человек открыл → тоже видит.

**Аналогия:**
- Без базы: блокнот который горит когда его закрываешь
- С базой: Google Docs — открываешь где угодно, всё на месте

**Что умеет база данных:**
- Хранить данные постоянно
- Делать запросы (найди все задачи со статусом «в работе»)
- Связывать данные между собой
- Работать с несколькими пользователями одновременно

## 3.5 Supabase: база данных без боли

Supabase — бесплатная база данных в облаке. Не нужен свой сервер. Не нужно знать SQL.

**Как работает:**
1. Регистрируешься на supabase.com → создаёшь проект
2. Создаёшь таблицу (как в Excel, но онлайн)
3. Claude Code пишет код который читает и записывает данные
4. В Supabase видишь что лежит в базе — как в таблице

**Пример для разных профессий:**

*Дизайнер:*
Таблица `briefs` — все брифы от клиентов. Добавил бриф → он сохранился → открыл через месяц → нашёл.

*UX researcher:*
Таблица `insights` — инсайты из интервью. Больше не теряются в папках.

*Архитектор:*
Таблица `projects` — все расчёты по объектам. История версий.

*CRM маркетолог:*
Таблица `segments` — сегменты базы. Можно смотреть динамику.

## 3.6 Fast Feedback Loop — быстрая обратная связь



> *«Теорема Найквиста говорит: чтобы контролировать систему, обратная связь должна работать минимум в два раза быстрее самой системы. AI генерирует код в 10 раз быстрее чем ты пишешь руками — значит проверять нужно тоже в 10 раз чаще.»*

**Практически:**

Представь двух поваров:

**Повар Изабелла:** пробует каждое блюдо перед тем как оно уйдёт к гостям. Если что-то не то — ловит сразу. Репутация звезды Мишлен.

**Повар Венсан:** блюда уходят непроверенными. Когда гости отравились — уже поздно. Проверяющий из Роспотребнадзора уже едет.

Вайбкодинг без проверки = повар Венсан. Ты можешь нагенерировать 500 строк сломанного кода прежде чем поймёшь что что-то пошло не так.

**Правило:** Сделал шаг → проверь результат → убедись что работает → следующий шаг.

## 3.7 Исследование конкурентов

Claude Code умеет открывать сайты, читать страницы, делать анализ.

**Промпт:**
```
Прочитай мой prd.md.
Найди 5-7 похожих продуктов (российский и международный рынок).
Для каждого: зайди на сайт, прочитай что предлагают.
Сделай таблицу: что есть у них, чего нет у меня.
Найди точки где я могу быть лучше.
Приводи ссылки на источники.
Сохрани в competitor-research.md
```

**Важно:** просить давать **ссылки на источники** — без ссылок AI может придумать несуществующие факты. С ссылками — можно проверить.

## 3.8 Как проверить результат AI

> *«Попроси AI проверить самого себя. Режим «критики» — это другой режим чем режим «генерации». Как повар который сначала готовит, а потом пробует — это два разных состояния.*

После того как получил анализ или скрипт:

```
Теперь проверь результаты которые только что дал.
Где могут быть ошибки или неточности?
Что нужно перепроверить вручную?
На чём я не должен полностью полагаться?
```

## 3.9 Визуализация: из цифр в понимание

После анализа — попроси красивый отчёт:

```
На основе файлов из папки results/
создай HTML-страницу с:
- ключевыми цифрами крупно (3-4 главных метрики)
- одним простым графиком
- тремя выводами что это значит
- двумя рекомендациями что делать
Сохрани в results/report.html
```

Занимает минуту у AI. Выглядит как профессиональный дашборд. Можно показать любому человеку.

## 3.10 Анализ своих данных — по профессиям

**🎬 Моушн дизайнер**
Что анализировать: история проектов (дедлайны, типы работ, клиенты).
Что искать: какие проекты самые маржинальные, какие занимают больше времени чем планировал.

**🔍 UX researcher**
Что анализировать: транскрипты интервью, паттерны ответов.
Скрипт: считает частоту упоминания тем, выделяет повторяющиеся паттерны.

**🎨 Дизайнер**
Что анализировать: история правок от клиентов.
Скрипт: находит что просят чаще всего, сколько итераций в среднем.

**✍️ Главред**
Что анализировать: статистика материалов (просмотры, дочитывания).
Скрипт: находит темы и форматы с лучшим вовлечением.

**📊 СМО**
Что анализировать: данные по каналам (ROAS, CPL, конверсии).
Скрипт: считает точные показатели по каждому каналу.

**🏗 Архитектор**
Что анализировать: история проектов, сметы, отклонения.
Скрипт: находит где чаще всего выходишь за бюджет.

**📧 CRM маркетолог**
Что анализировать: база подписчиков (открываемость, отток, LTV).
Скрипт: сегментирует по активности.

## Итого

| Концепт | Суть |
|---------|------|
| LLM и числа | Предсказывает, не считает → галлюцинирует |
| Python-скрипт | Детерминированный → всегда одинаковый результат |
| База данных | Данные живут в облаке, не в браузере |
| Supabase | Бесплатная база, без сервера |
| Fast feedback | Проверяй после каждого шага |
| Самопроверка AI | Попроси покритиковать своё же решение |

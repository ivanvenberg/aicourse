## Почему AI врёт уверенно

Это самое важное что нужно понять про работу с данными.

Из книги «Vibe Coding»:
> «Когда мы попросили AI посчитать NPS из одного файла — у разных людей получились разные цифры. Данные одни и те же. AI галлюцинировал у каждого по-разному.»

**Почему так происходит:**
Языковая модель — это не калькулятор. Она предсказывает слова. Она не считает — она угадывает правдоподобный ответ. И делает это с полной уверенностью.

Авторы описывают это так: *«AI ведёт себя как гений-самозванец: он всегда звучит убедительно, даже когда полностью неправ. Он объяснит тебе почему 2+2=5 таким образом, что ты почти поверишь.»*

**Правило:**
> Хочешь точные цифры → проси написать Python-скрипт → запускай скрипт → доверяй результату

Скрипт детерминирован: один и тот же код на одних и тех же данных всегда даёт один и тот же результат.

## Что такое база данных

До базы данных: пользователь добавил задачу → закрыл браузер → задача исчезла. Данные жили только в памяти браузера.

С базой данных: данные хранятся в облаке → открыл на телефоне через неделю → всё на месте.

**Аналогия:**
- Без базы: записная книжка которая сгорает каждый раз когда её закрываешь
- С базой: Google Docs — открываешь где угодно, всё на месте

---

## Что такое Supabase

Supabase — бесплатная база данных в облаке. Не нужен свой сервер. Не нужно знать SQL.

Ты регистрируешься → создаёшь таблицу (как в Excel) → Claude Code пишет код который читает и записывает данные.

**Почему Supabase:**
- Бесплатный план покрывает всё что нам нужно
- Claude Code умеет с ним работать из коробки
- Есть понятный интерфейс чтобы смотреть что лежит в базе

## Fast Feedback Loop — быстрая обратная связь

Из книги «Vibe Coding»:

> «Чем быстрее система работает — тем быстрее нужна обратная связь. Это как скорость машины: на 10 км/ч можно среагировать медленно. На 200 км/ч — нужны рефлексы гонщика.»

AI генерирует код в 10 раз быстрее чем ты пишешь руками. Значит ошибки тоже накапливаются в 10 раз быстрее. Значит проверять нужно так же часто.

**Практически:**
- Сделал шаг → проверь результат → убедись что работает → следующий шаг
- Не давай AI делать 10 вещей подряд без проверки
- Открывай браузер после каждого изменения

Авторы в книге называют это «taste every dish before it leaves the kitchen» — пробуй каждое блюдо перед тем как оно уйдёт к гостям.

## Почему нужна модульность

Из книги:

> «Исследования показали: разработчики которые работают в запутанных, немодульных системах в 9 раз чаще уходят или теряют работу. И AI работает так же плохо в запутанном коде.»

Модульность = когда код разбит на независимые части. Каждая часть делает одно дело.

**Почему это важно для вайбкодинга:**
- AI лучше справляется с маленькими конкретными задачами
- Если что-то сломалось — понятно где
- Можно развивать части независимо

**Как просить AI:**
```
Создай отдельный файл для логики работы с данными.
Держи интерфейс и бизнес-логику раздельно.
```

## Исследование конкурентов через Claude Code

Claude Code умеет открывать сайты, читать страницы, делать анализ.

**Промпт:**
```
Прочитай мой prd.md.
Найди 5-7 конкурентов для моего продукта.
Для каждого: прочитай сайт, посмотри что они предлагают.
Сделай сравнение: что есть у них, чего нет у меня.
Приводи ссылки на источники.
Сохрани в competitor-research.md
```

**Важно:** чем больше ссылок на источники — тем меньше галлюцинаций. Если нет ссылки — значит AI мог придумать.

## Визуализация данных

После анализа можно сделать красивый отчёт:

```
На основе файлов из папки results/
создай HTML-страницу с:
- ключевыми цифрами крупно
- графиками воронки
- главными выводами
- рекомендациями что делать
```

Это занимает минуту у AI и выглядит как профессиональный дашборд.

# Встреча 3 — Продукт который помнит. Настоящая база данных.

## Что сделаем

Добавим в продукт базу данных — чтобы данные хранились в облаке, а не в браузере. Это превращает учебный проект в настоящий рабочий инструмент.

Ещё каждый принесёт фидбек от живых людей — и мы разберём что улучшить.

## Структура встречи

**15 минут — фидбек-раунд**
Каждый рассказывает: показал продукт трём людям, вот что они сказали. Разбираем вместе.

**40 минут — Supabase + база данных**
- Что такое база данных и зачем она нужна (без страха)
- Регистрируемся в Supabase — бесплатная база в облаке
- Claude Code подключает базу к твоему продукту

**30 минут — работа над своим проектом**
Подключаем базу к каждому продукту. Данные теперь не исчезают.

**5 минут — домашка**

## Что такое Supabase

Supabase — это база данных в облаке. Бесплатно. Не нужен свой сервер.

Ты регистрируешься → создаёшь таблицу → Claude Code пишет код который с ней работает. Всё.

Примеры что это даёт:
- **Главред** → контент-план хранится в базе, не в голове или таблице
- **CRM маркетолог** → сегменты и письма сохраняются между сессиями
- **Архитектор** → все расчёты по проектам в одном месте
- **UX researcher** → транскрипты и инсайты не теряются

## Домашка

1. Покажи обновлённый продукт **трём живым людям** — не коллегам по курсу
2. Задай им три вопроса:
   - Что непонятно?
   - Что сломалось или не работает как ожидал?
   - Чего не хватает?
3. Запиши ответы — принеси на следующую встречу

Это важнее красивого кода. Настоящие пользователи всегда находят то, что ты не видишь.

# «Жизнь коротка — не откладывай радость на потом: пока горит твой миг, наполни его смыслом».  
— Омар Хайям

Привет, ребята. Здесь живут материалы наших встреч.

Зачем тут “Омар Хайям”: это наш **маячок обновлений**. Когда Ваня меняет что-то в репозитории — меняется и первая “цитата”. Увидел(а) новую цитату → значит, пора сделать `git pull`.

Важно: **не скачивайте “Download ZIP”**. Так вы получите “копию на сегодня” и потом не сможете нормально обновляться. Мы работаем через **git**.

## Установка (очень просто)

### 1) Поставь Cursor

Скачай и установи Cursor: **[cursor.com/download](https://cursor.com/download)**  
Запусти и создай аккаунт (Sign Up) — бесплатного плана хватит.

### 2) Поставь Git

Если Git уже стоит — пропусти шаг.

- **Mac**: открой Terminal и выполни `git --version`.  
  Если предложит поставить “Command Line Tools” — соглашайся.
- **Windows**: установи “Git for Windows”, потом открой Git Bash и выполни `git --version`.

### 3) Склонируй репозиторий (это важно)

Открой терминал и выполни:

```bash
git clone https://github.com/ivanvenberg/aicourse.git
cd aicourse
```

### 4) Открой проект в Cursor

В Cursor: **File → Open Folder** → выбери папку `aicourse`.

### 5) Как обновляться перед каждой встречей (1 команда)

Обновляйся:

- **перед каждой встречей**
- **и каждый раз, когда Ваня напишет в телеграме “обновитесь”**

Команда одна:

```bash
git pull
```

Если `git pull` пишет ошибки — просто пришли сюда текст ошибки (скопируй целиком).

## Если ты новый(ая)

1) Напиши, как тебя зовут (имя + фамилия).  
2) Проверь, что ты подготовился(ась) к первой встрече:
   - Cursor установлен и запускается
   - Репозиторий клонирован через `git clone`
   - `git pull` отрабатывает без ошибок
3) Мини-задание на фантазию (2–3 минуты):  
   **какой “идеальный проект” ты бы хотел(а) получить к концу курса?**  
   Запиши 3 пункта: что это, для кого, и почему это тебе важно.

Если мы не знакомы, вот простая проверка “по делу”: `verify.md`.

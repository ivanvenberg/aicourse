## Установка до первой встречи

### 1. Скачай Cursor

Cursor — это редактор кода с AI внутри. Именно в нём мы будем всё делать.

👉 **[cursor.com/download](https://cursor.com/download)**

Скачай, установи, запусти.

### 2. Создай аккаунт в Cursor

При первом запуске попросит войти. Нажми **Sign Up** и зарегистрируйся — это бесплатно.

> Бесплатного плана хватит для всего курса. Если хочешь больше — есть Pro за $20/мес, но это не обязательно.

### 3. Скачай этот репозиторий

Нажми зелёную кнопку **Code → Download ZIP** вверху страницы.

Распакуй архив в удобное место — например, на рабочий стол.

### 4. Открой папку в Cursor

- Запусти Cursor
- Нажми **File → Open Folder**
- Выбери папку `ai-course` (которую только что распаковал)

### 5. Проверь, что всё работает

Открой любой файл в Cursor и нажми:
- **Mac:** `Cmd + K`
- **Windows/Linux:** `Ctrl + K`

Должна появиться строка для AI-запроса. Если появилась — всё готово! 🎉

## Как ребятам быстро получать обновления

Самый удобный вариант — **не скачивать ZIP**, а один раз **клонировать репозиторий**. Тогда после каждого обновления вы подтягиваете изменения одной командой.

### Если вы ещё не скачивали проект

Откройте терминал и выполните:

```bash
git clone git@github.com:ivanvenberg/aicourse.git
cd aicourse
```

Дальше, когда тут появятся изменения, просто:

```bash
git pull
```

### Если вы уже скачали ZIP

ZIP сам по себе не “обновляется”. Самый быстрый переход на обновления:

- Сохраните свою работу (если вы уже что-то меняли)
- Склонируйте репозиторий как выше и дальше используйте `git pull`

## Куда загружать задания и файлы

Для заданий и любых материалов есть папка `uploads/`.

- **Куда класть**: `uploads/meeting-<номер>/<имя-фамилия>/...`
- **Пример**: `uploads/meeting-1/katya-gordeeva/README.md`

### Как сделать так, чтобы обновлялось “именно это”

Технически Git обновляет **весь репозиторий**. Но можно устроить процесс так, чтобы ребята меняли только `uploads/`, а остальное оставалось стабильным:

- **Рекомендуемый вариант (безопасно)**: включить защиту ветки `main` и принимать изменения **только через Pull Request**, где вы просите трогать только `uploads/`. Тогда у ребят будет привычный сценарий “залил(а) задание → открыл(а) PR”, а у тебя контроль.
- **Быстрый вариант (без PR)**: дать всем **write-доступ** — тогда они смогут пушить прямо в репозиторий (и обновления будут прилетать всем через `git pull`), но они смогут менять **любой** файл, не только `uploads/`.

Если хочешь, я напишу прямо сюда короткую пошаговую инструкцию “как ребятам отправлять PR с заданием”.

## Как общаться с ребятами (поддерживающе и вежливо)

Мы учимся вместе — вопросы и ошибки тут нормальны. Чтобы всем было комфортно:

- **Задавай вопросы смело**: если что-то не получается — так и пиши, это помогает быстрее разобраться.
- **Описывай контекст**: что ты хотел(а) сделать, что получилось, и что именно “сломалось” (можно скрин/текст ошибки).
- **Помогай без “умничанья”**: вместо “это же очевидно” — “давай покажу, как я бы сделал(а)”.
- **Критикуй мягко**: сначала что хорошо, потом что улучшить, и предложи следующий шаг.
- **Не перебивай и уважай темп**: у всех разная скорость; лучше уточнить, чем подгонять.
- **Давай место другим**: если уже ответили — можно добавить свой вариант, но без споров “кто прав”.
- **Береги личное**: не проси и не публикуй чужие пароли/ключи/личные данные; делись ссылками и файлами аккуратно.

Если не знаешь, как сформулировать — можно просто: “Ребята, помогите, пожалуйста: …”.

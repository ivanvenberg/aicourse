## 3. Теория: skills, агенты, проверка фактов

### Skills (команды / rules)

- **Skill** — короткая инструкция по «тегу» (например `/start`): повторяемый сценарий
- Закрывает: «он не пишет как я», «каждый раз делает по-разному»
- **≤50 строк** на skill — иначе не запомнит
- **Не один жирный rule** — несколько коротких rules
- Чтобы ИИ «писал как ты» — долго обучать, прописывать rules

### Проверка фактов

- LLM **всегда может нести чушь** — «классная ли идея?» = wooflo, ничего не значит
- Для поиска: **Parallel**, **Perplexity** — ходят по сайтам с citations, не «спроси ChatGPT»
- Маркетинговый анализ — фиксированный чеклист через skill, не импровизация

### Параллельные агенты-персоны

- Пока один агент работает — **другой может чистить файлы**, третий — другое
- Создаёшь **персоны** (MD-файлы): бизнесмен, маркетолог, PR — они **«ругаются»** друг с другом
- На выходе — более продуманный анализ, чем один ответ
- **Перед кодингом** — протестировать идею на 3–5 выдуманных «челиках»

### Контекстное окно

- **Вся книга не влезет** в контекст
- RAG: top-3 фрагмента, не «прочитай всё»
- Для точных ответов — минимальное окно + поиск по источникам
- **Notebook LM** (Google) — загрузка видео/лекций, вопросы по материалу

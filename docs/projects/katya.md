# Катя — Jira → Figma → Slack

## Зачем это тебе

Новый тикет в Jira → руками переносишь в Figma → пишешь в Slack. Хочется **автоматически**: тикет создали → **уведомление** → (потом) **фрейм в Figma** по шаблону.

Большая часть работы — **не код**, а согласовать поля в Jira и получить доступы. Это нормально.

---

## К чему идём (финал к 13 июня)

Jira → твой сервер → Slack (и Figma, когда plugin готов).  
На созвоне — **живое** или честный mock, если IT ещё не дал доступ.

## План на эту неделю

### 1. project.md (~30 мин)

Одна фича: «создали задачу в Jira → сообщение в Slack (+ очередь на Figma)».

---

### 2. Таблица полей (~1–2 ч + разговор с командой)

Файл `docs/jira-ticket-spec.md` — **один** тип тикета, **3–5 полей**:

| Поле Jira | Пример | Куда в Slack | Куда в Figma (позже) |
|-----------|--------|--------------|----------------------|
| key | BOSS-123 | заголовок | имя фрейма |
| summary | … | текст | — |
| month / project | … | канал / папка | page |

**Готово, когда:** ты или заказчик: «да, тикеты так заполняем».

---

### 3. Чеклист доступов (~30 мин)

Файл `access-checklist.md`:

- [ ] Jira webhook или Automation
- [ ] Slack incoming webhook / bot
- [ ] Figma token (для чтения; **создание** фрейма — через plugin, не REST)
- [ ] Vercel URL не блокируется IT

Красные галочки = **честно пишешь на созвоне**: «заблокировано, показываю mock».

---

### 4. Webhook Jira → Slack (~2–3 ч)

- `app/api/webhooks/jira/route.ts` на Vercel
- Тикет пришёл → Slack за **<5 сек** + сохранение в `logs/`
- В Slack пока: «Frame queued for BOSS-123» (Figma — **следующий этап**)

**Тест:** ngrok + реальный тикет **или** файл `fixtures/jira_issue_created.json` + curl.

**Готово, когда:** в Slack появилось сообщение с ключом задачи и ссылкой.

---

### 5. Git (~15 мин)

Секреты только в `.env`, не в репо.

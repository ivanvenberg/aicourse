# Юля — парсинг лидов (Telegram, потом LinkedIn)

## Зачем это тебе

В работе нужны **лиды в реальном времени**: кто-то в Telegram или LinkedIn пишет «ищу…», «нужен…» — ты хочешь это **ловить**, а не копать руками.

Второй запрос — вёрстка писем — **не кодим с нуля**: для этого есть Slava Bell, Framer и т.п. Сейчас фокус — **только лиды**.

---

## К чему идём (финал к 13 июня)

Скрипт или бот: **находит сообщения по твоим правилам** → каждая строка с **ссылкой на пост** → ты проверяешь глазами.

| Этап | Что |
|------|-----|
| **Сейчас** | Telegram: 2–3 **настоящих** лида + проверка руками |
| **Дальше** | LinkedIn — через **Apify** или другой API, если осознанно тратишь кредиты (см. ниже) |
| **Не сейчас** | Email-вёрстка, автоматические рассылки |

## Apify — если хочешь автоматизировать (не обязательно)

[Apify](https://apify.com/) — маркетплейс готовых scraper-ов («Actors»): Telegram, сайты, Google Maps и т.д. Cursor может подключить их через API.

| | |
|--|--|
| **Цена** | **Free plan: ~$5 usage credits каждый месяц**, карта **не нужна** |
| **Лимит** | Кредиты **не копятся** — сгорят к концу месяца; если $5 кончились — пауза до следующего цикла (без карты **не спишут** лишнее) |
| **Когда брать** | После **2–3 лидов руками** и manual-check — не вместо проверки, а чтобы не копипастить |
| **Что искать в Store** | `Telegram scraper`, `Telegram channel`, `Google Maps` — смотри **отзывы + цену за 1000 результатов** |

**Как не сжечь $5 за один вечер:**
1. Сначала **dry run** на 1 канале / 10 постов.
2. Выгрузка в JSON → тот же `filter_leads.py` + [`manual-check-template.md`](agents/manual-check-template.md).
3. Ключ Apify — только в `.env`, **не в git**.

**LinkedIn на Apify** — бывает, но дороже и капризнее; на неделе 1 **не надо**. Если пойдёшь позже — сначала оцени стоимость Actor в Store.

## План на эту неделю

### 1. Проект (~40 мин)

- Новый репо + `project.md` ([шаблон](../../lesson-4/project-template.md)):
  - **Кому:** ты, маркетинг / лиды
  - **Боль:** «пропускаю запросы в чатах»
  - **Фича:** «нашла пост по критериям → ссылка + текст»

**Готово, когда:** одной фразой объясняешь, что делает инструмент.

---

### 2. Правила «что такое лид» (~30 мин)

Файл `criteria.yaml` или `knowledge/lead-criteria.md`:

- **Плюс-слова:** «ищу», «нужен», «посоветуйте», «recommend»…
- **Минус-слова:** спам, не те вакансии…
- **10 примеров** «да, это лид» и «нет, не лид»

**Готово, когда:** изменила слово в правилах — другой пост проходит или нет.

---

### 3. Telegram (~2 ч)

**Вариант А (проще, 0 ₽):** файл `raw/messages.txt` — вручную 20–30 сообщений из каналов → `scripts/filter_leads.py` → `output/leads.json`.

**Вариант Б:** `scripts/telegram_fetch.py` (Telethon) — если уже умеешь, бесплатно.

**Вариант В (Apify, ~$5/мес free):** Actor из [Apify Store](https://apify.com/store) для Telegram → JSON в `raw/apify_export.json` → тот же фильтр. Сначала **маленький** прогон, потом manual-check.

- **3–5 каналов**, которые ты уже читаешь
- В каждой записи: канал, текст, **ссылка**, какие правила сработали

Не нужен «real-time каждую секунду» — **раз в день** или по кнопке достаточно для старта.

**Готово, когда:** в JSON **минимум 2–3** строки.

---

### 4. Проверка «не врёт» (~30 мин)

- Шаблон [`manual-check-template.md`](agents/manual-check-template.md) → `tests/manual-check.md`
- **3 строки:** нашла пост в Telegram руками → то же в JSON?

**Готово, когда:** у каждого лида есть **рабочая ссылка** на сообщение.

# Ира — планировщик меню + ВкусVill

## Зачем это тебе

Каждую неделю — голова занята: **что готовить на 4 человек**, быстро (до **20 минут в день**), из **ВкусVill у дома**, без подсчёта калорий.  
Инструмент должен **снимать ментальную нагрузку**, а не добавлять ещё одну «умную штуку, которая врёт про гребешки».

**Mealime** (совет Кати) — хороший пример, как удобно вести рецепты и списки. Твоя версия — **с ВкусVill и твоим адресом**.

---

## К чему идём (финал к 13 июня)

**Meal planner:** меню на неделю для семьи → продукты из ВкусVill → можно проверить и заказать.

Сейчас не всё сразу. Сначала — **честные данные**, потом — меню.

| Этап | Что получишь |
|------|--------------|
| **Сейчас (неделя 1)** | Список твоих продуктов → что ВкусVill **реально** отдаёт (✅ есть / ❌ нет / ❓ не уверена) |
| **Дальше** | Меню на несколько дней из твоих рецептов + найденных товаров |
| **Потом** | «По моему адресу можно заказать» — только если найдётся рабочий способ |

**Время на проект:** ориентир **до 5 часов в неделю** — у тебя менеджмент-период, это нормально.

---

## План на эту неделю

Делай **по порядку**. Не обязательно за один день.

### 1. Завести проект (~45 мин)

- Приватный репозиторий на GitHub (можно только для себя).
- Файл `project.md` — скопируй шаблон из [`lesson-4/project-template.md`](../../lesson-4/project-template.md) и заполни:
  - **Кому:** ты, семья 4 человека, ВкусVill, ≤20 мин готовки
  - **Боль:** «каждую неделю думаю, что готовить — утомляет»
  - **Что покажешь на этой неделе:** проверка списка продуктов без лжи

**Готово, когда:** открыла `project.md` — и за 30 секунд понятно, зачем проект.

---

### 2. Список продуктов (~30 мин)

- Файл `my-list.txt` — **10–15** продуктов, которые **реально покупаешь** (не «идеальный рацион»).
- Запиши район / адрес доставки (для будущей корзины).

**Готово, когда:** на каждый пункт можешь сказать «покупала на прошлой неделе».

---

### 3. Свои рецепты — текстом (~30 мин)

- Файл `knowledge/recipes-seed.md` — **5–10** рецептов ≤20 мин (просто текст, как ты готовишь).
- **Пока не подключай к коду** — это задел на следующий этап, но мотивация уже здесь.

**Готово, когда:** файл есть, можно открыть и прочитать.

---

### 4. Понять, откуда брать данные (~1 ч)

- Файл `research/notes.md` — коротко:
  - пробовала ли MCP / сайт ВкусVill;
  - что отдаёт (название, цена, ссылка);
  - **есть ли «остатки по адресу»** или только каталог.
- Один тестовый ответ сохрани в `research/vkusvill-raw.json`.

**Готово, когда:** в notes одна честная фраза: «по адресу могу / не могу проверить автоматически — потому что …».

**Важно:** если автоматически «по адресу» не получается — так и пиши. Лучше ❓ и «проверь в корзине», чем уверенная ложь.

---

### 5. Скрипт проверки списка (~1,5–2 ч)

Попроси Cursor (или сделай сама):

```
Прочитай project.md.
Python: scripts/vkusvill_check.py
Вход: my-list.txt
Выход: output/availability.json
Для каждого продукта: есть в каталоге (да/нет), ссылка на товар, кусок сырого ответа.
Не придумывать наличие — только то, что пришло из API/сайта.
```

**Готово, когда:** запустила скрипт → получила JSON по всем пунктам списка.

---

### 6. Проверка «не врёт» (~45 мин)

- Скопируй шаблон [`manual-check-template.md`](agents/manual-check-template.md) → `tests/manual-check.md`.
- Заполни **10 строк**: продукт → что видишь **руками на сайте** → что выдал скрипт → совпало?

**Готово, когда:** ≥8 из 10 совпали; 1–2 расхождения — ок, главное их **записала**.

---

### 7. Показать результат (~1 ч)

- Простой экран или команда в терминале: список → таблица ✅ ❌ ❓.
- Под таблицей текст: «Наличие по адресу — проверь в корзине ВкусVill» (если автоматом не умеешь).
- `.env` не коммить; `git push`.

**Готово, когда:** друг или ты с телефона — «понятно, что дальше» без длинных объяснений.

---

## Что не делать на этой неделе

- Меню на 7 дней × 4 человека
- Красивый дизайн, автозаказ
- Обещать «остатки по адресу», если источник этого не даёт

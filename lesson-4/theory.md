## 3.1 Что такое агент

**Агент** — AI-система которая выполняет многошаговые задачи используя инструменты.

**Чат-модель:**
```
Ты → запрос → Модель думает → Модель отвечает
```

**Агент:**
```
Ты → задача
  → думает → использует инструмент (читает файл)
  → видит результат → думает снова
  → использует другой инструмент (запускает код)
  → видит результат → думает снова
  → ... много шагов ...
  → сообщает результат
```

Ключевое: **цикл думать → действовать → наблюдать → думать снова**.

## 3.2 Архитектуры агентов

### ReAct (Reasoning + Acting)

Самый распространённый паттерн. Агент чередует рассуждение и действие.

```
Задача: «Найди ошибку в коде и почини»

Мысль: «Нужно прочитать файл»
Действие: читает main.js
Наблюдение: [содержимое файла]

Мысль: «Вижу проблему в строке 47»
Действие: исправляет строку
Наблюдение: «Файл изменён»

Мысль: «Нужно проверить что работает»
Действие: запускает тест
Наблюдение: «Тест прошёл»

Ответ: «Исправил ошибку...»
```

**Почему работает:** каждое действие основано на реальном наблюдении, не на предположении.

---

### Plan-and-Execute

Сначала составляет весь план, потом выполняет по шагам.

```
Задача: «Создай контент-пайплайн»

Фаза 1 — план:
→ [1. скилл tone-of-voice, 2. скилл find-news, 3. агент-копирайтер]

Фаза 2 — выполнение:
→ создаём по плану
```

**Когда лучше ReAct:** не знаешь заранее что понадобится.
**Когда лучше Plan-and-Execute:** понятная структура, большой проект.

## 3.3 Инструменты агентов и как каждый ломается

Агент без инструментов — советник без рук.

### Файловая система
Читает, создаёт, изменяет, удаляет файлы.
**Как может сломаться:** удалил важный файл без предупреждения (реальная история — потеряли 10 000 строк кода).
**Защита:** `git commit` перед каждой большой операцией.

### Терминал
Запускает программы, устанавливает пакеты, делает запросы.
**Как может сломаться:** команда работает на Mac но не на Windows; бесконечный цикл; зависимость не установлена.
**Защита:** проверяй окружение перед запуском.

### Браузер
Открывает сайты, кликает, читает.
**Как может сломаться:** CAPTCHA, авторизация, динамический контент.
**Защита:** используй официальные API когда возможно.

### API
Запросы к внешним сервисам.
**Как может сломаться:** rate limiting, изменился формат ответа, кончились деньги.
**Защита:** обрабатывай все ошибки явно.

## 3.4 Мульти-агентные системы

Несколько агентов работают параллельно.

**Команда дебатёров:**
```
Генератор → предлагает идеи для улучшения продукта
Адвокат   → защищает каждую идею (почему хорошая)
Критик    → атакует каждую идею (почему плохая)
→ консенсус который пережил проверку
```

**Параллельные исследования:**
```
Агент 1: анализирует сильные стороны продукта
Агент 2: анализирует слабые стороны
Агент 3: исследует конкурентов
→ все работают одновременно
→ собираем результаты
```

**Проблема контентии:** несколько агентов изменяют один файл → конфликты. Решение: каждый агент работает в отдельном файле.

## 3.5 Декомпозиция задач

Если задача слишком большая — агент теряет суть, срезает углы, «забывает» требования.

**Граф задач:**
```
Создать контент-пайплайн
├── Скилл: tone-of-voice
│   ├── Прочитать мои тексты
│   └── Сохранить профиль стиля
├── Скилл: find-news
│   ├── Открыть источники
│   └── Сохранить список
└── Агент: copywriter
    ├── Читает tone-of-voice
    ├── Читает список новостей
    └── Пишет пост
```

**Правило:** начинай с минимально работающей версии. Добавляй по одной функции, каждый раз проверяя.

## 3.6 Контекстные файлы: долгосрочная память

AI не помнит ничего между сессиями. Контекстные файлы дают ему «память».

**CLAUDE.md** — правила проекта (Claude Code читает автоматически):
```markdown
## Стек
- Frontend: HTML + CSS + JS
- База: Supabase

## Правила
- Комментарии только если код неочевиден
- Никогда не удаляй файлы — перемещай в /archive/

## Запрещено
- Устанавливать пакеты без разрешения
- Менять схему базы без согласования
```

**MEMORY.md** — информация о тебе:
```markdown
## Кто я
Главред медиа. Аудитория — менеджеры 30-45 лет.
Тон: умный, без занудства.

## Мой продукт
Инструмент для контент-планирования.
```

**AGENTS.md** — разрешения:
```markdown
## Разрешения
- MAY: читать/редактировать /src/
- MAY NOT: коммитить в main напрямую
- MAY NOT: изменять .env файлы
## При неопределённости → спрашивай перед действием
```

## 3.7 Три проблемы агентов

### «Проблема с детьми»
Агент делает видимость выполнения вместо реального решения. Вместо починки теста — отключает его. Вместо решения проблемы — убирает проверку.

**Защита:**
```
После задачи проверь явно:
- сколько тестов было? сколько сейчас?
- тесты реально тестируют что должны?
```

### «Картонные кексы»
Агент хардкодит значения чтобы тесты «проходили». Выглядит как работающий код, внутри — пустышка.

**Защита:** смотри не только что тесты зелёные, но что они реально проверяют.

### «Слизняк-мусорщик»
После сессии остаются: debug-логи везде, переменные `interim_result5`, закомментированный код, временные файлы.

**Защита:**
```
После завершения задачи:
Удали все debug-логи, неиспользуемые переменные и закомментированный код.
```

## 3.8 Скиллы: переиспользуемые промпты

**Скилл** — сохранённый промпт, вызывается через `/`.

**Место:**
```
.claude/skills/[название]/skill.md
```

**Формат:**
```markdown
---
name: brief-generator
description: Создаёт бриф после интервью с клиентом
---

Задай клиенту 7 вопросов по одному. Жди ответа.
[вопросы]
После → создай структурированный бриф. Сохрани в briefs/
```

| Профессия | Скилл | Что делает |
|-----------|-------|-----------|
| Главред | `/контент` | Тема → структура → черновик |
| Дизайнер | `/бриф` | Интервью → структурированный бриф |
| СМО | `/отчёт` | Данные → еженедельный отчёт |
| Архитектор | `/смета` | Параметры → расчёт стоимости |
| CRM | `/письма` | Сегмент → варианты писем |
| Моушн | `/сторибород` | Бриф → детальный сторибород |
| UX | `/инсайты` | Транскрипт → структурированные инсайты |

## 3.9 MCP: реальные инструменты для агентов

**MCP** — протокол подключения к внешним сервисам.

Без MCP: AI читает и пишет файлы.
С MCP: AI открывает Figma, читает Google Sheets, создаёт задачи в Notion.

```bash
claude mcp add figma    # дизайнеры, моушн
claude mcp add gdrive   # главреды
claude mcp add gsheets  # СМО, CRM, архитектор
claude mcp add gmail    # СМО, CRM
claude mcp add notion   # UX, главреды
```

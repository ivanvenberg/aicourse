## Контекстное окно — рабочий стол AI

Из книги «Vibe Coding»:

> «Представь стол. Ты раскладываешь на нём материалы для работы. Чем больше бумаг — тем меньше места и сложнее ориентироваться. Когда стол переполнен, помощник начинает складывать старые бумаги в стопку — и важные детали теряются.»

Claude держит в памяти ~200 000 токенов. Это много, но не бесконечно.

**Что происходит когда контекст переполняется:**
- Срабатывает `compact` — Claude сжимает историю
- Он может «забыть» важные детали
- Может нарушить инструкции которые давал раньше
- Авторы книги описывают случай: они сказали Claude «не делай X», потом произошёл compact, и он сделал X — потому что забыл

**Правила:**
- Открывай новую вкладку при каждой новой задаче
- Следи за индикатором — `/context` в Claude Code
- После 70% заполнения — новый чат

## Скиллы: сохранённые промпты

Скилл — это промпт который сохранён в файле и вызывается одной командой.

**Где хранятся:**
```
.claude/
  skills/
    tone-of-voice/
      skill.md    ← промпт
    find-news/
      skill.md
```

**Как вызвать:** напиши `/tone-of-voice` — Claude Code прочитает промпт и выполнит.

**Формат skill.md:**
```markdown
---
name: tone-of-voice
description: Анализирует тексты и выделяет стиль автора
---

Прочитай все тексты в папке texts/.
Выдели:
- характерные слова и фразы
- структуру текстов
- тон и подачу
- что автор никогда не пишет
Сохрани как описание стиля в voice-profile.md
```

**Зачем:**
Вместо того чтобы каждый раз писать длинный промпт — один раз создал скилл и вызываешь его одним словом.

## Агенты: параллельные помощники

Из книги «Vibe Coding»:

> «Представь что у тебя есть задача: купить квартиру, машину и найти стоматолога. Ты можешь делать это последовательно — или запустить трёх помощников параллельно, каждый занимается своим. Каждый работает в своей «комнате» и не мешает другим.»

**Чем агент отличается от скилла:**
- Скилл: промпт в текущем контекстном окне
- Агент: отдельный процесс, своя память, работает параллельно

**Как запустить:**
```
Запусти 3 субагента параллельно:
1. Проанализируй сильные стороны моего продукта (prd.md)
2. Найди слабые стороны
3. Предложи 5 идей для улучшения
Каждый сохраняет результат в отдельный файл.
```

**Когда использовать агентов:**
- Когда задача большая и делится на независимые части
- Когда хочешь несколько перспектив одновременно
- Когда нужно исследовать несколько направлений параллельно

## MCP: руки для Claude

**MCP (Model Context Protocol)** — протокол который позволяет Claude подключаться к внешним сервисам.

Из книги:
> «Без MCP AI — это умный советник без рук. С MCP — он может открыть Figma, прочитать Google Sheets, отправить письмо. Это фундаментальный шаг: от «расскажи мне что делать» к «сделай это сам».»

**MCP под твою профессию:**

| Профессия | MCP | Что делает |
|-----------|-----|-----------|
| Дизайнер | Figma | Читает макеты, описывает, предлагает правки |
| Главред | Notion + Google Docs | Пишет прямо в твои документы |
| СМО | Google Sheets + Gmail | Читает метрики, отправляет отчёты |
| Архитектор | Google Sheets | Заполняет сметы |
| CRM маркетолог | Google Sheets + Gmail | Сегментирует базу, пишет письма |
| UX researcher | Notion | Сохраняет инсайты напрямую |
| Моушн дизайнер | Figma | Читает фреймы, пишет сторибород |

**Установка:**
```bash
claude mcp add figma
claude mcp add gdrive
claude mcp add gsheets
claude mcp add gmail
```

## MEMORY.md: долгосрочная память

Файл который Claude читает при каждом запуске. Ему не нужно объяснять кто ты и что делаешь.

**Создай в корне проекта:**

```markdown
# Кто я

[Имя], [профессия].
Работаю в [тип компании/индустрия].

# Мой продукт

[Одно предложение что делает продукт]

# Важные правила

- [Что важно]
- [Что нельзя делать]
- [Предпочтения]

# Контекст

[Любая информация которую важно помнить]
```

Это как инструкция новому сотруднику — один раз написал, больше не объясняешь.

## Команда агентов-дебатёров

Из книги «Vibe Coding»:
> «Мы создали команду из трёх агентов: Генератор придумывает идеи, Адвокат защищает каждую, Критик атакует. В итоге получаешь не просто список идей — а идеи которые пережили спор.»

Это экспериментальная фича. Включается через:
```
Добавь в настройки Claude Code переменную:
CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

После этого можно создавать команды агентов которые разговаривают между собой.

## Контент-пайплайн: система из скиллов

Для главредов и всех кто создаёт контент — это главное упражнение урока.

**Как устроен:**

```
Скилл tone-of-voice: анализирует твои тексты → профиль стиля
         ↓
Скилл topic-map: анализирует темы → карта тем
         ↓
Скилл find-news: читает сайты → список новостей за сегодня
         ↓
Агент copywriter: берёт новость + стиль → пишет пост
         ↓
Готовые черновики постов в твоём стиле
```

Одна команда `/посты` → система делает всё сама.

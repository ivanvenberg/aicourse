## 3.1 Четыре уровня работы с AI

```
Уровень 1 — Чат          Спросил → ответил. Файлы копируешь сам.
Уровень 2 — Редактор     Агент правит файлы в проекте (Cursor).
Уровень 3 — Субагент     «Сходи, сделай X, верни результат».
Уровень 4 — Агент+MCP    Руки в Notion, Sheets, браузере, поиске.
```

**AI-native** — ты живёшь на уровнях 2–4: задача словами, результат в файлах/сервисах, ты проверяешь.

**Agent Teams** (несколько сессий Claude Code, которые пишут друг другу) — эксперимент для терминала. **Не обязательны.** На курсе достаточно субагентов и MCP.

---

## 3.2 Агент: цикл «думаю → делаю → смотрю»

**Чат:**
```
Ты → вопрос → модель → текст
```

**Агент (ReAct — Reason + Act):**
```
Ты → задача
  → «надо прочитать brief.md»     → читает файл
  → «тут нет API ключа»           → просит .env
  → «запущу скрипт»               → терминал
  → «тест упал»                   → правит код
  → «готово»                      → отчёт
```

Каждый шаг опирается на **реальный** результат прошлого шага, не на фантазию.

**Пример промпта:**
```
Прочитай data/reviews.json.
Напиши Python-скрипт для подсчёта средней оценки.
Запусти. Если ошибка — исправь и запусти снова.
Сохрани результат в results/avg.md
```

## 3.3 Субагенты

**Субагент** — отдельный запуск агента на подзадачу. Результат возвращается **тебе** (или главному агенту), не другим субагентам.

**Когда использовать:**
- три независимых анализа (без спора между ними);
- параллельно: «один читает код, второй пишет тесты, третий README».

**Пример:**
```
Запусти 3 субагента параллельно:
1. Найди все места где вызывается API
2. Найди уязвимости (ключи в коде, нет .env)
3. Предложи 3 улучшения архитектуры
Каждый → отдельный файл в audit/
Потом сведи в audit/summary.md
```

**Не путать с театром в одном чате** («сыграй генератора и критика») — это одна голова, не субагенты.

## 3.4 Tool use (function calling)

Модель может не только писать текст, но и **вызывать функции**:

```
Пользователь: «Какая погода в Москве?»

Модель решает: нужен tool weather(city)
→ вызывается реальная функция
→ результат: -2°C, снег
→ модель: «В Москве -2°C, снег»
```

В Claude Code tools встроены: Read, Write, Bash, Grep…  
В MCP — tools внешних сервисов.

**Ты проектируешь агента**, когда решаешь: какие tools ему доступны и какие **запрещены**.

---

## 3.5 MCP — «руки» агента

**MCP (Model Context Protocol)** — стандарт: один раз подключил сервис → агент умеет им пользоваться.

| Без MCP | С MCP |
|---------|--------|
| Только файлы в папке | Notion, Figma, Google Sheets, Gmail… |
| Ты копируешь из сервиса | Агент читает/пишет сам |

**Установка (в папке проекта):**
```bash
claude mcp add notion
claude mcp add gsheets
claude mcp add figma
```

Токен — в настройках сервиса, не в чате.

**Пример задачи:**
```
Через Notion MCP: прочитай страницу «Интервью 12.03»,
выдели 5 цитат, сохрани в insights.md
```

**Parallel MCP** (поиск в интернете для агентов) — см. §3.8.

## 3.6 Скиллы

**Скилл** — сохранённая инструкция, вызов одной командой `/имя`.

Путь: `.claude/skills/имя/skill.md`

```markdown
---
name: weekly-report
description: Собирает недельный отчёт из папки data/
---

1. Прочитай все CSV в data/
2. Запусти scripts/summary.py
3. Сохрани отчёт в reports/YYYY-MM-DD.md
4. Формат: цифры крупно, три вывода
```

Вызов: `/weekly-report`

**Скилл = твой рабочий процесс в тексте.** Чем больше повторяешь — тем больше скиллов.

## 3.7 Память проекта

| Файл | Что писать |
|------|------------|
| `CLAUDE.md` | Стек, стиль кода, запреты («не удаляй файлы») |
| `MEMORY.md` | Кто ты, проект, аудитория, тон |
| `AGENTS.md` | Права: MAY / MAY NOT |
| `.env` | API-ключи — **никогда** в git и не в промпте |

**Пример CLAUDE.md:**
```markdown
## Стек
HTML + JS, Supabase, OpenRouter

## Правила
- Коммит перед большой правкой
- Секреты только в .env
- Не удалять — переносить в /archive/
```

Агент читает это при старте — не объясняешь контекст заново.

## 3.8 Поиск в интернете: Perplexity и Parallel

Модель **не знает** свежие новости и **выдумывает** ссылки. Для фактов из сети — **поисковый API**.

### Perplexity API

**Зачем:** ответ с **источниками**, меньше галлюцинаций на фактах.

**Когда:** «что изменилось в законе», «какие есть аналоги сервиса», быстрый ресёрч.

**Паттерн:**
```
1. Запрос → Perplexity API
2. Ответ + citations (URL)
3. Ты или агент проверяешь ссылки
4. Итог в research.md
```

Ключ: `PERPLEXITY_API_KEY` в `.env`.  
Документация: [docs.perplexity.ai](https://docs.perplexity.ai)

**Пример (идея, не копируй ключи в код репо):**
```python
# scripts/ask_perplexity.py — ключ из os.environ
# «Какие MCP-серверы популярны для Notion в 2026?»
# → сохранить ответ + список URL в research/mcp-notion.md
```

### Parallel API

**Зачем:** поиск и обогащение данных **для агентов** — выдержки со страниц, структурированный ресёрч.

| API | Что делает |
|-----|------------|
| **Search** | «Найди всё про X» → отрывки с URL |
| **Task** | Вход (таблица, компания) → структурированный JSON на выходе |

Ключ: `PARALLEL_API_KEY` в `.env`.  
Документация: [docs.parallel.ai](https://docs.parallel.ai)

**Search — пример запроса (curl):**
```bash
curl https://api.parallel.ai/v1/search \
  -H "Content-Type: application/json" \
  -H "x-api-key: $PARALLEL_API_KEY" \
  -d '{
    "objective": "Найти свежие статьи про RAG для маленьких команд",
    "search_queries": ["RAG small team 2026", "vector database tutorial"]
  }'
```

**Task — идея:** на входе `{ "company": "Notion" }`, на выходе JSON с полями по твоей схеме.

**В Claude Code:** Parallel можно подключить как MCP (`parallel-web`) — агент сам вызывает поиск.

### Когда что

| Задача | Инструмент |
|--------|------------|
| Быстрый ответ с ссылками | Perplexity |
| Много страниц, выдержки для RAG | Parallel Search |
| Таблица → дополнить колонками из сети | Parallel Task |
| Код в твоём проекте | Агент + файлы |
| Точные цифры по твоим данным | Python-скрипт (урок 3) |

## 3.9 Декомпозиция задач

Большая задача в один промпт → агент теряет половину.

**Плохо:**
```
Сделай полноценный SaaS с оплатой и админкой
```

**Хорошо — граф:**
```
1. Схема БД (schema.md)
2. API endpoints (api.md)
3. Одна страница UI
4. Подключить .env
5. Деплой
```

Один шаг → проверка → следующий. **AI-native** = ты держишь архитектуру, агент крутит шаги.

## 3.10 Три ловушки агентов

| Ловушка | Что делает | Защита |
|---------|------------|--------|
| **Дети** | Отключает тесты вместо починки | «Сколько тестов было/стало?» |
| **Кексы** | Хардкод под тест | Читай код, не только «passed» |
| **Мусор** | debug-логи, `temp2` | «Убери мусор после задачи» |

**Всегда:** `git commit` перед тем как отпустить агента на большую правку.

---

## 3.11 Контекст агента

Длинный чат → модель «забывает» начало → режет углы.

**Три приёма:**
1. **Новый чат** на новую задачу  
2. **Файлы-сводки** (`context-summary.md`)  
3. **CLAUDE.md** — правила не обсуждаются каждый раз  

```
Суммаризируй сессию: что сделали, решения, что осталось.
Сохрани в context-summary.md
```

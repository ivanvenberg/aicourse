## 3.1 Можно продать — без бизнес-плана

Пять вопросов в `brief.md`:

1. **Кто платит?** — конкретная роль или имя  
2. **За что?** — экономия времени, меньше ошибок, не нанимать человека  
3. **Сколько стоит боль?** — часы × деньги  
4. **Почему не ChatGPT в браузере?** — удобство, данные, процесс  
5. **Кто сказал «куплю»?** — цитата не из курса  

Красные флаги: «для всех», «AI для саморазвития», нет ни одного живого собеседника.

---

## 3.2 Лендинг-оффер + app = одна ссылка

```
https://твой-продукт.vercel.app/
├── /      ← кому, боль, цена, кнопка «Попробовать»
└── /app   ← инструмент (одна платная фича)
```

Покупатель не должен искать «а где сам продукт».

---

## 3.3 Агент

**Чат:** запрос → ответ.

**Агент:** думает → действует (файл, терминал, API) → смотрит результат → снова думает.

**ReAct** — чередование рассуждения и действия на основе **реального** наблюдения, не фантазии.

**Субагенты** — несколько задач параллельно, каждый пишет в свой файл → потом сводка.

---

## 3.4 MCP — руки агента

Без MCP агент только в папке проекта.  
С MCP — Notion, Sheets, Figma, Gmail и т.д.

```bash
claude mcp add notion   # UX, главреды
claude mcp add gsheets  # СМО, CRM
claude mcp add figma    # дизайн, моушн
claude mcp add gdrive   # тексты
claude mcp add gmail    # рассылки
```

Не встало на уроке — в домашку. Скилл важнее для старта.

---

## 3.5 Скиллы

Папка `.claude/skills/имя/skill.md` → вызов `/имя`.

Один скилл = одна еженедельная рутина вокруг продукта: `/бриф`, `/отчёт`, `/письма`.

---

## 3.6 Память проекта

| Файл | Содержание |
|------|------------|
| `CLAUDE.md` | Стек, правила кода, запреты |
| `MEMORY.md` | Кто ты, продукт, тон, покупатель |

Агент читает при старте — не объясняешь контекст каждый раз.

---

## 3.7 Три ловушки агентов

1. **«Дети»** — отключил тест вместо починки. Спрашивай: тесты реально проверяют поведение?  
2. **«Кексы»** — захардкодил ответ под тест. Смотри код, не только зелёные галочки.  
3. **«Мусор»** — debug-логи, мёртвый код. Проси убрать после задачи.

**Страховка:** `git commit` перед большой правкой агента.

## Контекстное окно

Claude держит в памяти 200 000 токенов — это примерно 150 000 слов. Когда память заполняется, происходит **compact**: Claude сжимает историю разговора и может потерять важные детали.

**Что делать:**
- Открывай новую вкладку при каждой новой задаче
- Следи за индикатором заполнения в нижнем углу
- Команда `/context` — сколько процентов занято

## Скиллы

Скилл — это сохранённый промпт который вызывается одной командой.

**Где хранится:**
```
.claude/
  skills/
    tone-of-voice/
      skill.md
    find-news/
      skill.md
```

**Как вызвать:** напиши `/tone-of-voice` в Claude Code.

**Зачем нужен:**
Вместо того чтобы каждый раз писать длинный промпт — один раз создал скилл и вызываешь его одним словом.

**Формат skill.md:**
```markdown
---
name: tone-of-voice
description: Анализирует тексты и выделяет стиль автора
---

Прочитай все тексты в папке texts/.
Выдели:
- характерные слова и фразы
- структуру текстов
- тон и подачу
- что автор никогда не пишет
Сохрани как подробное описание стиля.
```

## Агенты (субагенты)

Агент — это Claude который запускается параллельно в отдельном контекстном окне.

**Чем отличается от скилла:**
- Скилл: один промпт, вызывается через /слэш, работает в текущем окне
- Агент: отдельный процесс, своя память, может работать параллельно с другими

**Когда использовать агентов:**
Когда задача большая и её можно разбить на части которые делаются одновременно.

Пример:
```
Запусти 3 субагента параллельно:
1. Найди новости про AI в дизайне за сегодня
2. Найди новости про AI в редактуре за сегодня  
3. Найди новости про AI в маркетинге за сегодня
Каждый сохраняет результат в свой файл.
```

## MCP — руки для Claude

MCP (Model Context Protocol) — это способ подключить Claude к внешним сервисам.

Без MCP: Claude только читает и пишет файлы на твоём компьютере.
С MCP: Claude открывает Figma, читает Google Sheets, отправляет письма.

**Как установить:**
```bash
claude mcp add figma
claude mcp add gdrive
claude mcp add gmail
```

**MCP под твою профессию:**

| Профессия | MCP | Что делает |
|-----------|-----|-----------|
| Дизайнер | Figma | Читает макеты, описывает, предлагает правки |
| Главред | Notion + Google Docs | Пишет напрямую в твои документы |
| СМО | Google Sheets + Gmail | Читает таблицы, отправляет письма |
| Архитектор | Google Sheets | Заполняет сметы |
| CRM маркетолог | Google Sheets + Gmail | Читает базу, пишет письма |
| UX researcher | Notion | Сохраняет инсайты напрямую |
| Моушн | Figma | Читает фреймы и пишет сторибород |

## MEMORY.md

Файл который Claude читает при каждом запуске. Это его долгосрочная память о тебе.

Создай в корне проекта файл `MEMORY.md`:

```markdown
# О проекте и владельце

## Кто я
[Имя], [профессия], работаю в [тип компании/индустрия].

## Мой продукт
[Одно предложение что делает продукт]

## Важные правила
- [Что важно для меня в работе]
- [Что нельзя делать]
- [Предпочтения по стилю]

## Контекст
[Любая информация которую важно помнить между сессиями]
```

После этого не нужно каждый раз объяснять кто ты и что делаешь.

## Команды агентов (Agent Teams)

Три агента которые спорят между собой:
- **Генератор** — придумывает идеи для продукта
- **Адвокат** — защищает каждую идею
- **Критик** — атакует каждую идею

В результате получаешь не просто список идей, а проверенные идеи с аргументами за и против.

Включается через переменную в настройках:
```
Добавь в настройки Claude Code переменную CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

## Блок 1. Выбор продукта (25 мин)

*Теория: §3.0 «можно продать»*

1. Скопируй `brief-template.md` → в репо продукта как `brief.md`
2. Заполни. Нет идеи → `ideas-fallback.md`
3. **Холодное сообщение:** напиши одному знакомому (не из курса):

```
Привет! Делаю [одна фраза из brief].
Сколько бы ты заплатил за это / взял бы на работу? Честно.
```

4. Вставь ответ в `brief.md` → поле «Доказательство»

✅ `brief.md` готов + есть цитата или «напишу до конца дня» → **+ в чат**

**К перерыву без brief — садим в пару к тому, у кого есть идея.**

---

## 🔥 Упражнение: «60 секунд за деньги»

В парах. Один — «покупатель», другой — «фаундер».

**60 секунд:** только из `brief.md` — кому, боль, цена. Без слов «нейросеть», «AI», «курс».

Покупатель может сказать только:
- «Понял, купил бы»
- «Не понял, за что плачу»
- «Дорого / дёшево — потому что…»

Меняетесь. Один инсайт в чат.

## Блок 2. Лендинг-оффер + скелет app (25 мин)

*Теория: §3.10 лендинг + app в одном репо*

```
Прочитай brief.md.

Сделай в одном проекте:
1. Главная / — лендинг-оффер: кому, боль, что получит, цена, кнопка «Попробовать»
2. /app — заглушка продукта (одна платная фича — следующий блок)
3. CLAUDE.md и MEMORY.md из brief и моей профессии
4. Задеплой на Vercel. Дай ссылку.
```

✅ Одна ссылка, лендинг открывается → **+ в чат**

---

## Блок 3. Одна платная фича (25 мин)

Только то, за что в `brief.md` названа цена. Не десять кнопок.

```
Прочитай brief.md.
Реализуй ОДНУ фичу в /app — ту, за которую покупатель платит.
Подключи Supabase если нужно хранить данные.
Проверь сам как покупатель: с лендинга → в app → результат за < 2 минут.
```

✅ Фича работает по ссылке → **+ в чат**

## Блок 4. Скилл (15 мин)

*Теория: §3.8 скиллы*

```
Из brief.md: какая рутина вокруг этого продукта бесит каждую неделю?
Создай скилл одной командой. Сохрани в .claude/skills/[название]/skill.md
Проверь: /[название] запускается.
```

Примеры: `/бриф`, `/пост`, `/отчёт`, `/письма`

✅ Скилл отработал один раз на встрече → **+ в чат**

## Блок 5. MCP (15 мин)

*Теория: §3.9 MCP*

Выбери под профессию:

```bash
claude mcp add figma    # дизайн, моушн
claude mcp add notion   # UX, главреды
claude mcp add gsheets  # СМО, CRM, архитектор
claude mcp add gdrive   # главреды
claude mcp add gmail    # СМО, CRM
```

```
Через MCP сделай одно действие для МОЕГО продукта из brief.md
(прочитай таблицу / файл / макет — и результат в проект).
```

Не встало? **Не стопорим урок** — MCP в домашку, скилл уже есть.

✅ MCP сработал ИЛИ «домашка: MCP» → **+ в чат**

## 🔥 Упражнение по комнатам: «Битва агентов за фичу»

*Теория: §3.2 ReAct vs Plan-and-Execute, §3.4 мульти-агент*

Одна комната — один `brief.md` (с разрешения автора).

**Команда А — Plan-and-Execute:**
```
Составь план: как улучшить платную фичу этого продукта.
Выполни план. Сохрани в team-a.md
```

**Команда Б — 3 субагента параллельно:**
```
1. Что сломает покупатель в первые 5 минут
2. Что добавить чтобы заплатили в 2 раза больше
3. Одна правка UI за 10 минут кода
→ файлы team-b1.md, b2.md, b3.md + summary.md
```

5 минут: чьи идеи полезнее для **продажи**, не для красоты?
